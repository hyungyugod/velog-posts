# 📌 1. 엑셀 데이터 입력 원리 

## 1-1. 문자 데이터(Character Data)

문자 데이터는 “숫자를 포함하더라도 계산 불가능한 데이터"를 말한다.  
문자, 기호, 숫자가 섞여도 엑셀은 **문자열**로 간주한다.

### 기본 정렬
왼쪽 정렬 → 문자.

### 특징
- 문자+숫자 혼합 → 문자로 인식  
- 특수문자 포함 → 문자  
- 전화번호, 주민번호, 코드값 → 문자  
- 숫자를 문자로 입력: `'123`  
- 줄바꿈: Alt+Enter  
- 일본어가 나오는 이유: ㄸ + 한자키 → 일본어·특수문자 목록  
- 한자 변환: 글자/단어 선택 후 한자키

---

## 1-2. 숫자 데이터(Number Data)

계산 가능한 모든 데이터.

### 정렬
오른쪽 정렬.

### 사용 가능한 기호
+, -, (), %, E/e, .

### 숫자 인식 예
1,234 → 숫자  
1.23E+10 → 숫자  
(100) → 음수 표시  

### 분수 입력
분수는 날짜와 충돌하기 때문에 **0 + 공백** 필요.  
예: `0 1/2` → 0.5 로 인식.

### 11자리 이상
지수(E) 형태로 자동 변환.

---

## 1-3. 날짜(Date) / 시간(Time) 데이터

### 날짜는 일련번호
1900-01-01 = 1  
1900-01-02 = 2  
2025-01-01 = 44927

### 시간은 하루의 분수
12:00 = 0.5  
06:00 = 0.25  

### 날짜+시간
날짜와 시간 사이 공백 필수.  
예: 2025-01-01 13:20

### 단축키
Ctrl + ; = 오늘 날짜  
Ctrl + Shift + ; = 현재 시간

---

## 1-4. 수식 데이터

= 또는 + 로 시작.

수식 표시: Ctrl + ~

# 📌 7. 텍스트/날짜 함수 개념

## LEFT(B1,1)
B1의 “내용”에서 왼쪽 1글자.

## TRIM
Trim = 다듬다 (공백 제거)

## PROPER
Proper = 고유명사 표기 (단어 첫 글자 대문자)

## EDATE
E = Elapsed → “월 단위 경과 날짜”

## SUBSTITUTE(text, old, new, n)
n번째 old_text만 new_text로 치환.  
빈값("")이면 “삭제”.

예: SUBSTITUTE(A1,"-","",5)  
→ A1의 5번째 '-'만 삭제.

# 📌 10. LOOKUP 계열 완전 정리

## 10-1. LOOKUP(배열 방식)

LOOKUP(value, array)

- 2D 배열 → 첫 행에서 검색, 마지막 행에서 반환  
- 세 번째 인수 **무시**  
- 항상 **근사값 탐색**

## 10-2. VLOOKUP

VLOOKUP(찾을값, 표, 열번호, TRUE/FALSE)

TRUE → 근사값  
FALSE → 정확 일치

### 근사값 규칙 (TRUE)
찾는 값 ≤ 표의 값 중 가장 큰 값  
(lower bound, 마지막 하한값)

---

## 10-3. HLOOKUP

가로 방향 검색.  

HLOOKUP(value, table, row_index, TRUE/FALSE)

- row_index = 실제 반환할 행  
- 장식 아님  
- LOOKUP과 절대 혼동 금지

---

## 10-4. CHOOSE + VLOOKUP

CHOOSE({1,2}, B:B, A:A)  
= 가상 2열 테이블 생성  
1열 = B  
2열 = A

---

## 10-5. MATCH / XMATCH

MATCH(value, range, 0)  
0 = 정확 일치  
1 = 이하 최대  
-1 = 이상 최소  

XMATCH는 확장판.

---

## 10-6. INDEX

INDEX(범위, 행, 열)  
MATCH와 조합 → VLOOKUP 대체

---

# 📌 11. 근사값(True)의 수학적 규칙

lookup_value ≤ x 중  
가장 큰 값 선택.

$$match = \max(x \mid x \le lookup\_value)$$

그 x가 속한 행/열의 값을 col_index_num 또는 row_index_num에 따라 반환.

---

# 📌 12. 실전 문제 

## 12-1. LOOKUP 43 문제

lookup_value = 43  
첫 행: 10, 30, 50, 70  
43 ≤ 값 중 최대 → 30  
30의 위치 = 두 번째  
결과 행(B3:E3)의 두 번째 → 400

---

## 12-2. HLOOKUP “전기세”

HLOOKUP("전기세", A1:C5, 2, FALSE)

- 첫 행에서 “전기세” 찾음 (C1)  
- 2번째 행 반환 → C2 = 32000

---

## 12-3. VLOOKUP 101-301 TRUE

A열 오름차순:  
101-201  
101-202  
101-301  
101-302

근사값(TRUE)  
→ 찾는 값 ≤ 값 중 최대 = 101-301  
3열 값 → 29000
