
# 📌 1. 화이트박스 테스트와 제어구조 검사

## 1-1. 화이트박스 테스트 개요
화이트박스 테스트(White-box Testing)는 프로그램의 내부 구조, 코드 로직, 제어 흐름을 기반으로 테스트를 설계하는 방법이다.  
즉, **코드 내부를 직접 확인하면서 테스트**하는 구조적 테스트다.

### 주요 종류
- **제어구조 검사(Control Structure Testing)**: 제어 흐름(조건문, 반복문 등)을 기반으로 테스트.
- **데이터 흐름 검사(Data Flow Testing)**: 변수의 정의와 사용 경로를 점검.
- **루프 검사(Loop Testing)**: 반복문을 다양한 반복 횟수로 테스트.
- **조건 검사(Condition Testing)**: 복합 조건식의 모든 조합을 테스트.

---

# 📌 2. 제어구조 검사의 세부 개념

## 2-1. 문장 커버리지
- 모든 문장이 한 번 이상 실행되도록 테스트.  
예: if 문 안의 문장이 실행되지 않으면 문장 커버리지가 부족.

## 2-2. 결정 커버리지(Decision Coverage)
- 모든 분기(if, switch 등)의 **True/False** 결과가 최소 한 번씩 실행되도록 테스트.

예시:
```c
if (x > 0)
    y = 1;
else
    y = 2;
```
- x > 0 (True) → y=1  
- x <= 0 (False) → y=2  
=> 두 케이스 모두 실행되어야 결정 커버리지 100% 달성.

## 2-3. 경로 커버리지(Path Coverage)
- 가능한 모든 실행 경로를 최소 한 번 이상 테스트.  
- 가장 강력하지만 현실적으로 모든 경로를 테스트하기 어려움.

---

# 📌 3. 기초경로 테스트(Basis Path Testing)

- 프로그램의 **독립 경로(Independent Path)** 를 최소 한 번 이상 수행하도록 하는 화이트박스 기법.
- 제어 흐름 그래프(Control Flow Graph)와 **순환복잡도(Cyclomatic Complexity)** 를 이용.

공식:  
$V(G) = E - N + 2$  
- $E$: 간선 수 (Edges)  
- $N$: 노드 수 (Nodes)

또는 간단히:  
$V(G) = P + 1$  (P: 분기문의 개수)

$V(G)$ 값이 높을수록 코드 복잡도 증가 → 테스트 케이스도 많아진다.

# 📌 4. 정보보안의 3요소 (CIA Triad)

| 요소 | 영문 | 설명 | 예시 |
|------|------|------|------|
| **기밀성** | Confidentiality | 인가되지 않은 사용자는 정보에 접근 불가 | 암호화, 접근제어 |
| **무결성** | Integrity | 정보가 불법적으로 변경되지 않음 | 해시, 전자서명 |
| **가용성** | Availability | 언제나 접근 가능한 상태 유지 | 백업, 이중화, DDoS 방어 |

> 즉, **비밀은 지키고, 내용은 변하지 않으며, 언제든 접근 가능해야 한다.**

# 📌 5. 내부버스와 외부버스

## 5-1. 내부버스 (Internal Bus)
- CPU와 메모리 등 **내부 구성요소 간 통신 통로**.
- 종류:
  - **데이터 버스**: 실제 데이터 이동.
  - **주소 버스**: 메모리 주소 전달.
  - **제어 버스**: 읽기/쓰기 신호, 제어 명령 전달.

## 5-2. 외부버스 (External Bus)
- CPU 외부의 **주변장치와 통신하는 버스**.
- 예: USB, PCIe, SATA, I/O 버스.

정리:  
내부버스는 CPU 내부, 외부버스는 주변장치 간 통신.

---

# 📌 6. Time Slice(타임 슬라이스)

- 운영체제의 CPU 스케줄링에서 프로세스에 **CPU를 할당하는 시간 단위**.  
- 라운드로빈(Round Robin) 스케줄링에서 각 프로세스는 지정된 타임 슬라이스 동안만 CPU 사용.

목적:  
- 모든 프로세스가 CPU를 **공평하게 사용**하게 함.  
- 대화형 시스템의 **응답성을 향상**시킴.

---

# 📌 7. CIDR과 DHCP

## 7-1. CIDR (Classless Inter-Domain Routing)
- IP 주소 낭비를 줄이기 위해 A/B/C 클래스 구분 없이 서브넷 비트로 네트워크를 표현.

### 철자 분석
- **C**: Classless  
- **I**: Inter  
- **D**: Domain  
- **R**: Routing  

예시: `192.168.10.0/24`  
→ 255.255.255.0  
→ 네트워크 비트 24, 호스트 비트 8.

## 7-2. DHCP (Dynamic Host Configuration Protocol)
- IP 주소를 자동으로 할당해주는 프로토콜.

### 철자 분석
- **D**: Dynamic (동적)  
- **H**: Host (호스트)  
- **C**: Configuration (설정)  
- **P**: Protocol (프로토콜)

### 동작 과정 (DORA)
1. Discover – 클라이언트가 서버를 탐색.  
2. Offer – 서버가 IP 제안.  
3. Request – 클라이언트가 요청.  
4. Acknowledge – 서버가 승인.

# 📌 8. TKIP과 IPsec

## 8-1. TKIP (Temporal Key Integrity Protocol)
- WEP의 보안 취약점을 보완하기 위해 만든 **무선 LAN 보안 프로토콜**.
- **패킷마다 키를 변경**하여 재사용 공격 방지.
- WPA(Wi-Fi Protected Access) 표준에서 사용되며 WPA2에서는 AES-CCMP로 대체.

## 8-2. IPsec (Internet Protocol Security)
- IP 계층(Layer 3)에서 암호화·무결성·인증을 제공하는 **보안 프로토콜 집합**.
- VPN 통신의 핵심 기술.

| 구분 | IPsec | TKIP |
|------|--------|------|
| 계층 | 네트워크 계층 | 데이터링크 계층 |
| 용도 | IP 전체 통신 보호 | 무선 LAN 데이터 보호 |
| 환경 | 유선/무선 공통 | Wi-Fi 한정 |
| 후속 기술 | 계속 사용 | WPA2의 AES로 대체 |

# 📌 10. 그레이웨어(Greyware)

## 10-1. 개념
화이트웨어(정상 프로그램)와 블랙웨어(악성코드)의 **중간 단계**.  
사용자의 동의 없이 설치되거나, 광고·정보 수집 등으로 불쾌감을 유발.

## 10-2. 종류

| 종류 | 설명 |
|------|------|
| 애드웨어(Adware) | 광고를 자동으로 띄움 |
| 스파이웨어(Spyware) | 사용자 행동·정보 추적 |
| 트랙웨어(Trackware) | 웹 활동을 추적해 광고용 분석 |
| 툴바웨어(Toolbarware) | 브라우저에 툴바 설치 |
| 다운로더(Downloader) | 다른 프로그램 자동 설치 |
| 점퍼웨어(Jumperware) | 임의 사이트로 강제 이동 |
| 스크린세이버웨어 | 내부적으로 광고·정보수집 수행 |
| 조크웨어(Jokeware) | 장난성 프로그램, 겁주는 목적 |
| 피싱웨어(Phishingware) | 로그인창 위조, 개인정보 유출 유도 |

# 📌 12. 레지스터, 플립플롭, 래치, CPU 구성과 IPv6, 관계대수

## 12-1. 플립플롭(Flip-Flop)
- 1비트의 정보를 저장하는 **기억 소자**.
- 입력에 따라 출력이 0 또는 1로 유지됨.
- **클록(Clock)** 신호에 맞춰 상태가 변하는 **동기식 회로**.
- 종류: RS, D, JK, T 플립플롭 등.
- 디지털 회로에서 **기억 기능의 기본 단위**로 사용됨.

---

## 12-2. 래치(Latch)
- 플립플롭보다 단순한 **비동기식 기억소자**.
- 입력 신호가 변하면 즉시 출력이 변함.
- 속도는 빠르지만 안정성이 떨어짐.

| 구분 | 래치(Latch) | 플립플롭(Flip-Flop) |
|------|--------------|----------------------|
| 동작 방식 | 비동기식 | 동기식(클록 기반) |
| 출력 변화 시점 | 입력 변화 시 즉시 | 클록 상승/하강 에지에서 |
| 용도 | 간단한 임시 저장 | 레지스터, CPU 회로 구성 |

---

## 12-3. 레지스터(Register)
- **여러 개의 플립플롭으로 구성된 저장장치**로 여러 비트를 동시에 저장.
- CPU 내부에서 연산 중간결과, 명령어, 주소 등을 저장.
- 일반적으로 8비트, 16비트, 32비트, 64비트 단위로 구성됨.

### 주요 역할
- 연산의 입력 및 결과값 임시 저장
- 명령어 실행에 필요한 주소/데이터 보관
- 메모리 접근 시 필요한 정보 임시 유지

---

## 12-4. CPU 구성 요소와 레지스터 위치
CPU는 크게 **제어장치(CU)** 와 **연산장치(ALU)** 로 구성된다.

| 구성 요소 | 약자 | 역할 |
|------------|------|------|
| 제어장치 | CU (Control Unit) | 명령어 해석, 제어 신호 생성 |
| 산술논리연산장치 | ALU (Arithmetic Logic Unit) | 산술/논리 연산 수행 |
| 레지스터 | Register | 연산 중간값, 명령어, 주소 저장 |

### 주요 레지스터 종류
- **ACC (Accumulator)**: 연산 결과 저장  
- **IR (Instruction Register)**: 현재 명령어 저장  
- **PC (Program Counter)**: 다음 명령어 주소 저장  
- **MAR (Memory Address Register)**: 접근할 메모리 주소 저장  
- **MBR (Memory Buffer Register)**: 읽기/쓰기 데이터 임시 저장  

정리하면, 레지스터는 **CPU 내부의 초고속 임시 기억공간**이며 플립플롭으로 구성된다.

---

## 12-5. IPv6 구조
- IPv6는 **128비트 주소 체계**이며, **16비트씩 8부분**으로 나뉜다.
- 각 16비트는 16진수 4자리로 표현되고, 콜론(`:`)으로 구분된다.

예시:
```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```
- 앞쪽은 네트워크 식별, 뒤쪽은 호스트 식별.
- 연속된 0은 `::` 로 한 번만 생략 가능.

예시(축약형):
```
2001:db8:85a3::8a2e:370:7334
```

| 구분 | IPv4 | IPv6 |
|------|------|------|
| 주소 길이 | 32비트 | 128비트 |
| 표현 방식 | 10진수 4부분 | 16진수 8부분 |
| 브로드캐스트 | 있음 | 없음(애니캐스트로 대체) |

## 12-6. 관계대수(Relational Algebra)에서 합집합과 조인

| 연산 | 개념 | 결과 | 전제 조건 |
|------|------|------|-----------|
| **합집합 (Union)** | 두 릴레이션의 모든 튜플을 합침 (중복 제거) | 동일한 속성 구조의 릴레이션 | 속성 개수와 도메인이 동일해야 함 |
| **조인 (Join)** | 공통 속성을 기준으로 두 릴레이션을 결합 | 새로운 릴레이션 생성 | 공통 속성이 존재해야 함 |

### 예시
- 합집합: 학생A(1,2,3), 학생B(3,4,5) → 결과 (1,2,3,4,5)  
- 조인: 학생(학번,이름) ⋈ 성적(학번,점수) → (학번, 이름, 점수)

**정리:**  
- 합집합은 **같은 구조의 데이터를 합치는 것**,  
- 조인은 **서로 다른 테이블을 결합해 새로운 정보를 만드는 것**이다.
