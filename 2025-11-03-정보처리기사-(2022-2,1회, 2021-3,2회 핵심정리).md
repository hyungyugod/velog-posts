# 📌 1. 대칭키 암호 & ARIA
## 1-1. 대칭키 개요
- 동일한 비밀키로 암·복호화.
- 장점: 빠름 / 단점: 키 배포·관리 어려움.

## 1-2. ARIA
- 한국 표준 대칭키 블록암호(국정원 주도). 경량·하드웨어·소프트웨어 구현 모두 고려.
- 블록 크기 128비트, 키 길이 128/192/256비트.

# 📌 2. SQL 다중행 비교 연산자
## 2-1. IN / SOME(ANY) / ALL / EXISTS
- IN: 집합에 “포함”.
- SOME(=ANY): “하나라도” 조건 만족. 예) $x > \text{SOME}(S)$ 는 $x$가 $S$의 최소값보다 크면 참.
- ALL: “모두” 조건 만족. 예) $x > \text{ALL}(S)$ 는 $x$가 $S$의 최대값보다 커야 참.
- EXISTS: 서브쿼리 결과 존재 여부만 검사(조건식은 상관 서브쿼리에서 자주 사용).

# 📌 6. 목적기반 테스트

# 📌 7. 부분/완전 함수 종속
## 7-1. 개념과 2NF
- 부분 함수 종속: 복합키의 “일부”에 종속(2NF 위반).
- 완전 함수 종속: 복합키 “전체”에만 종속.
- 2NF: 1NF + 부분 종속 제거.

# 📌 10. RAID 요약
## 10-1. 수준별
- RAID0(스트라이핑): 성능↑, 내결함성 없음.
- RAID1(미러링): 안전↑, 용량효율 50%.
- RAID5(분산 패리티): 성능·안전 균형, 1디스크 장애 허용.
- RAID6(이중 패리티): 2디스크 장애 허용.
- RAID10(1+0): 성능+안전↑, 최소 4디스크.

# 📌 11. TKIP / WEP / 무결성
## 11-1. WEP
- RC4 + 24bit IV + CRC-32(오류검출). 고정키·짧은 IV로 취약.
## 11-2. TKIP
- RC4 유지하되 패킷별 키 믹싱, 48bit IV, MIC 도입(WPA의 과도기).
## 11-3. 무결성(Integrity)
- 전송 중 변조 탐지 보장. CRC는 오류검출용, 암호학적 MAC/MIC는 위조 탐지 가능.

- ISMS-P는 개인정보보호 요구사항 포함.

# 📌 15. 나눗셈 연산자 비교
## 15-1. C/Java/Python
- C/Java: `/` 하나뿐. 정수/정수는 몫(소수 버림), 실수 포함시 실수 나눗셈.
- Python: `/` 실수 나눗셈, `//` 내림(floor) 몫. 예) $-5 // 2 = -3$.

# 📌 16. Java의 `^` 와 비트 연산
## 16-1. `^`는 XOR
- “다르면 1”. 제곱 아님. 제곱은 `Math.pow`.
- 유사군: `&`(AND), `|`(OR), `~`, 시프트 `<<`, `>>`, `>>>`.

# 📌 17. 불식 예제 평가
## 17-1. 결과
- `a=3,b=4,c=3,d=5` 일 때  
  `(a==2 | a==c)` → `true`, `!(c>d)` → `true`, `(1==b ^ c!=d)` → `true`  
  전체: `true & true & true` → **true**.  
- `|`, `&` 사용 시 단락 평가 없음.

# 📌 18. `&`의 이중성
## 18-1. 타입에 따른 의미
- 정수: 비트 AND.  
- boolean: 논리 AND(단락 평가 없음).  
- 단락 평가가 필요하면 `&&` 사용.

# 📌 19. 링크 계층 vs 전송 계층(오류/흐름 제어)
## 19-1. 범위 차이
- 데이터 링크(2계층): **노드 간** 프레임 수준(CRC, ARQ, 흐름제어, MAC).
- 전송(4계층): **종단 간** 세그먼트 수준(TCP 재전송, 윈도우, 포트).

# 📌 20. Factory Method vs Abstract Factory
## 20-1. 구분 핵심
- Factory Method: “어떤 구상 제품을 만들지 **서브클래스**가 결정”(상속·오버라이드, 단일 제품 축).
- Abstract Factory: “**제품군**을 일관되게 생성”(합성, 여러 생성 메서드).

# 📌 21. ARP 스푸핑
## 21-1. 원리/대응
- 거짓 ARP 응답으로 IP→MAC 매핑 오염 → MITM/DoS.  
- 대응: 스위치 DAI+DHCP Snooping, 포트 보안, 정적 ARP(핵심 장비), TLS/VPN, VLAN 분리.

# 📌 22. 럼바우 모델링(OOA)
## 22-1. 세 모델
- 객체 모델(Object/Information): 클래스·속성·관계(정적 구조).  
- 동적 모델: 상태도/전이(행위·시간).  
- 기능 모델: DFD(입력→처리→출력 흐름).

# 📌 23. Information vs Object 용어
## 23-1. 의미
- Information: 업무상 다루는 개념/데이터적 실체.  
- Object: 그 정보를 시스템 내 객체로 추상화.  
- 럼바우에서 “Information Model ≈ Object Model”로 병칭.

# 📌 24. 정규형 요약 & Ad-hoc
## 24-1. 정규형 한눈표
- 1NF: 원자값.  
- 2NF: 부분 함수 종속 제거.  
- 3NF: 이행 종속 제거.  
- BCNF: 결정자는 모두 후보키.  
- 4NF: 다중값 종속 제거.  
- 5NF: 조인 종속 제거.
## 24-2. Ad-hoc
- “즉석의, 그때그때” — Ad-hoc 쿼리/분석/네트워크: 사전 정의 없이 필요 시 수행·구성.

# 📌 25. 병행제어 기법
## 25-1. 요약
- 잠금(2PL/Strict 2PL): 직렬화 보장, 데드락 관리 필요.
- 낙관적 검증(OCC): 충돌 적은 환경에 효율.
- 타임스탬프(TO/Thomas Rule): 순서 강제·오래된 쓰기 무시.
- MVCC/Snapshot Isolation: 읽기 비차단, 쓰기 충돌·write skew 주의.
- 격리수준: RU/RC/RR/SERIALIZABLE — 일관성 vs 성능 트레이드오프.
