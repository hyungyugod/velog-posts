# 📌 1. 함수와 레이블

## 1-1. Swift 함수 레이블 (Argument Label)

> 💡 Swift는 함수 호출 시 레이블을 붙여 "읽히는 코드"를 만드는 것이 기본 설계 철학이다.

### 1) 기본 레이블 구조

- Swift 함수는 기본적으로 외부 레이블(argument label) + 내부 파라미터명을 분리해서 설계된다.
- 호출 시 레이블을 함께 써야 하며, 자연어처럼 읽히는 코드가 목표다.
- 레이블 없이 호출하면 컴파일 오류가 발생한다.

```swift
func move(from start: Int, to end: Int) { }

move(from: 3, to: 7)
// 읽으면 바로 이해됨 → "3에서 7로 이동해"
```

### 2) 레이블 생략 (_)

- 파라미터 앞에 `_`를 붙이면 외부 레이블을 생략하고 호출할 수 있다.
- 주로 짧은 유틸리티 함수나 수학 연산처럼 문맥이 명확할 때 사용한다.
- 프로그래머스 등 코딩테스트 자동 생성 함수 시그니처에서 자주 등장한다.

```swift
import Foundation

func solution(_ num1: Int, _ num2: Int) -> Int {
    return num1 * num2
}

solution(3, 5)  // 레이블 없이 호출
```

### 3) 반환 타입 (->)

- `->` 는 함수가 반환하는 값의 타입을 명시한다.
- 반환값이 없으면 `-> Void` 또는 생략한다.

```swift
func add(_ a: Int, _ b: Int) -> Int {
    return a + b
}
```

### 4) 정수 나눗셈과 타입 변환

- Swift의 `/` 연산자는 Int끼리 연산하면 소수점을 버리고 몫만 반환한다.
- 실수 결과를 얻으려면 두 수 중 하나 이상을 `Double`로 형변환해야 한다.

```swift
let a = 7 / 2           // 3  (소수점 버림)
let b = Double(7) / 2   // 3.5 (실수 연산)
```
