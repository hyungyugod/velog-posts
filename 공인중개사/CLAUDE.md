# 공인중개사 폴더 — Claude 작업 규칙

## 📌 OX 퀴즈 오답 정리 루틴 (필수)

사용자가 OX 퀴즈를 푼 뒤 **"정리해줘 / 오답 정리 / 틀린 것만 정리"** 등 정리 요청을 하면, 다음 루틴을 자동으로 따른다.

### 1. 저장 위치
- 폴더: `/Users/hg/Desktop/velog-posts/공인중개사/claude_ox_오답/`
- 폴더가 없으면 먼저 생성한다.

### 2. 파일명 규칙
- 형식: `YYYY-MM-DD-claude_oxquize.md`
- 예시: `2026-05-20-claude_oxquize.md`
- **같은 날짜의 OX 퀴즈는 같은 파일에 이어서 누적 저장**
  - 이미 파일이 있으면 새로 만들지 말고 **append (이어쓰기)**
  - 회차 번호를 붙여 구분 (예: `## 🟨 3회차 오답`, `## 🟩 4회차 오답`)
  - 회차 안의 문제 번호는 원래 출제 번호 그대로 유지 (예: 5번, 12번)

### 3. 저장할 내용
오답에 한해서만 다음 구조로 정리:
- **문제** (원문 그대로)
- **정답** (O 또는 X)
- **해설** (왜 그런지, 핵심 개념, 비교표 등)
- **함정 키워드** (어떤 함정에 걸렸는지)
- **암기 팁** (있으면)

### 4. 마무리 섹션 (선택, 양이 충분할 때만)
- 패턴 분석표 (어떤 약점에 자주 걸리는지)
- 핵심 암기카드 (5장 내외)

### 5. 정리 후 컨펌
- 파일 저장 후 computer:// 링크로 공유
- "오늘 누적 X문제 정리 완료" 식으로 요약 한 줄

## 📌 OX 퀴즈 출제 시 기본 원칙

- 사용자가 OX 퀴즈 요청 시 → 먼저 **범위·문항 수·난이도** 3개를 AskUserQuestion으로 확인
- 문제는 사용자의 실제 노트(공인중개사 폴더 내 `.md` 파일들) 내용에서 출제
- 정답키는 outputs 폴더에 별도 보관 (사용자에게 노출 X)
- 답안 받으면 정·오답 표 + 틀린 것만 상세 해설
- 마지막에 약점 패턴 분석 (의심병·숫자 함정·이름 함정 등) 한 줄로 요약

## 📌 콘텐츠 변환 관점 (사용자 작동 방식 반영)

- 오답 정리는 단순 답안지가 아니라 **카드뉴스·블로그 변환 가능한 형태**로 작성
  - 비교표 적극 활용
  - 암기코드(예: 일일계, 4321) 강조
  - 시각적으로 분리되는 구조 (이모지 헤더, 표, 굵은 글씨)
- 함정 키워드는 매번 명시 — 패턴 누적 분석에 사용됨
