# 주간 오답 수거 지침 v3 (2026-07-02 개정 — Fable 설계 WO-1·WO-7)

> 스케줄러 `weekly-wrong-answer-collection`에 적용된 프롬프트의 사본. 수정하려면 이 파일을 고친 뒤 "이 파일로 스케줄러 지침 갱신해줘"라고 요청.
> **v3 변경점:** 집계를 node 스크립트 → `build_ledger.py`(원장 v2)로 교체, Downloads 접근 실패해도 계속 진행(원장이 전체 이력이라 무손실), **기대치 검증**(생성된 퀴즈 수 vs 도착한 결과 수 대조 — 무음실패 백신) 신설, 노트를 원장 뷰(졸업·상습·듀 큐·콘텐츠 소재 후보)로 재정의.
> 롤백: `ai-guidelines/Fable5_설계요청/_백업_스케줄프롬프트_2026-07-02/` + 이 폴더의 `오답노트_지침_v2.md`.

---

매주 월요일 실행되는 "주간 오답 수거 + 원장 갱신" 작업이다.

**[실행 원칙] 오답 데이터는 결과 JSON에 있는 것만 쓴다(지어내기 금지). 집계는 손으로 하지 않고 build_ledger.py가 한다. 접근이 안 되면 추측하지 말고 그대로 보고한다.**

## 0. 경로·중복 가드 (가장 먼저)
- bash `date +%F`로 오늘 날짜를 구한다.
- `ROOT=$(find /sessions/*/mnt -maxdepth 2 -type d -name velog-posts 2>/dev/null | head -1)` — 비어 있으면 "velog-posts 마운트 없음 — 수거 불가"라고 명확히 보고하고 종료.
- `BASE=$ROOT/공인중개사/claude_ox_오답`. `$BASE/{오늘날짜}-claude_oxquize.md`가 이미 있으면 "이번 주 오답노트는 이미 생성되어 있다"고만 알리고 종료.

## 1. 수거 (Downloads → _inbox)
- `ls /sessions/*/mnt/`에서 Downloads류 마운트를 찾아 `find <마운트> -maxdepth 1 -name '공인중개사_오답_*.json' -mtime -8` → 각 파일을 `cp -n`으로 `$BASE/_inbox/`에 복사한다. 브라우저 사본 `... (1).json`도 그대로 복사(원장이 알아서 사본 제거).
- Downloads 접근 불가여도 **종료하지 않는다** — "Downloads 미연결: _inbox/_raw 기존분만으로 진행"이라고 명시하고 계속한다(원장은 전체 이력 누적이라 늦게 도착해도 손실 없음).

## 2. 원장 갱신 (집계의 단일 원천)
- `python3 "$BASE/_ledger/build_ledger.py" --ingest` 실행.
- exit 2(원천 0건)면 그 출력을 그대로 보고하고 종료. 다른 오류도 숨기지 말 것.
- 갱신된 `$BASE/_ledger/오답_원장.json`을 읽는다. **이후 모든 수치·목록은 이 파일에서만 가져온다.**

## 3. 기대치 검증 (무음실패 백신 — 반드시 수행)
- 지난 7일 생성된 퀴즈: `find "$ROOT/공인중개사/데일리퀴즈" -name '20*.html' -mtime -8` (템플릿 `_template.html`은 제외 패턴상 자동 제외됨).
- 각 퀴즈 `{스템}.html`에 대해 `공인중개사_오답_{스템}*.json`이 `_raw` 또는 `_inbox`에 존재하는지 대조한다.
- 결과가 없는 퀴즈가 있으면: "⚠️ 결과 미도착 N개: {스템 목록} — 미제출이거나 다운로드 유실. 해당 퀴즈를 다시 제출하거나, 제출 화면의 '📋 결과 JSON 복사'로 _inbox에 넣어달라"를 **채팅 보고와 노트 양쪽에** 기재한다. 조용히 넘어가지 말 것.

## 4. 주간 오답노트 생성 (원장 뷰)
- 저장: `$BASE/{오늘날짜}-claude_oxquize.md`. "지난주" = 원장 이벤트 중 날짜 접두(YYYY-MM-DD)가 지난 7일에 속하는 것.
- 구성(순서대로):
  1. 제목 + 인용블록: 기간 · 제출 횟수 · 평균 · 지난주 오답 수 (원장 통계 사용)
  2. `## 🟦 과목명` 섹션 — 지난주 오답 각 문항: `### N. 개념 제목` 아래 **문제 / 정답 / 해설 / 함정 키워드**. 2회 이상 틀린 개념은 제목에 `🔁 N회 틀림`을 붙이고 과목 안 맨 위로. (원재료는 지난주 JSON의 wrong/results와 원장 samples — 여기 없는 오답을 추가하거나 있는 오답을 빠뜨리지 말 것)
  3. `## 🎓 이번 주 졸업·졸업후보` — 원장 status가 졸업/졸업후보인 개념 (없으면 "아직 없음 — v2 결과가 쌓이면 시작된다" 한 줄)
  4. `## 🔴 상습 경보` — 원장 상습 목록 표
  5. `## 🗓️ 다음 주 복습 큐` — 원장 dueQueue 상위 8개 표 (이게 다음 퀴즈 재도전 원천이다)
  6. `## 📊 패턴 분석` — 지난주 오답 5분류 표(숫자 함정 / 의심병 / 주체·권한 혼동 / 단정 표현 / 비슷한 이름 쌍) + 가장 큰 적신호 1개와 대응 1줄
  7. `## 🔑 핵심 암기카드 5장` — 지난주 오답에서 한 줄 암기 5개(숫자·암기코드 중심)
  8. `## 💡 콘텐츠 소재 후보 3` — 지난주 오답·듀 큐에서 카드뉴스 각이 서는 개념 3개, 각각 **질문형 훅 1줄**과 함께 (예: "전세금은 왜 돌려받기 어려울까?")
  9. `## ✅ 기대치 검증` — §3 결과 (전부 도착 or 미도착 목록)
- 지난주 오답이 0개(전부 만점)면 2·6·7은 생략하고 통계 + 졸업 소식 + 검증 결과만 담는다.

## 5. 마무리
- 이번에 새로 처리한 JSON 베이스네임을 `$BASE/_processed.txt`에 append (참고 이력 — 원장 자체는 멱등이라 중복 집계 걱정 없음).
- 완성 노트를 present_files로 전달하고, 약한 과목·상습·소재 후보를 두 문장으로 요약한다. 모든 문구 한국어. 추천 질문 3개 포맷은 생략.
