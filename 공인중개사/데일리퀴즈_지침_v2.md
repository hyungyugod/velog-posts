## 2. 출제 계획표 작성 (문항 작성 전 필수 — 건너뛰기 금지)

문항을 쓰기 전에 40행 계획표를 먼저 만든다(형식: `# | 과목 | 출처 소제목 | 개념 + 함정 설계(무엇을 한 끗 바꿀지) | 유형 | 정답idx | 🔁여부`).

### 2-1. 과목 배분
- 40문항을 발견된 과목 수로 고르게 나눈다(과목 간 차이 ≤1, 재도전이 몰린 과목만 +2 허용). 예: 6과목이면 7·7·7·7·6·6.
- **QUESTIONS 배열에서 같은 과목 문항은 연속 배치한다**(과목이 바뀌는 지점에서만 섹션 헤더가 생긴다).

### 2-2. 유형 쿼터 (합계 40)

| 유형 | 문항 수 | 비고 |
|---|---|---|
| 4지선다 일반형 | 20 | "…옳은 것은?" / "…<b>옳지 않은</b> 것은?" |
| 4지선다 ㄱㄴㄷ 조합형 | 7 | 발문에 정확히 "모두 고른 것은?" 포함 |
| 4지선다 계산형 | 4 | 탄력성·세액·동의율·면적·기간 등. 소재 부족 시 최소 2, 부족분은 일반형으로 |
| 4지선다 사례형 | 4 | 갑·을·병이 등장하는 사례에 법리 적용 |
| OX | 5 | 빠른 개념 확인용. OX도 한 끗 함정(주체·숫자·방향)을 심는다 |

- 4지선다 35문항 중 **부정형("옳지 않은/해당하지 않는/아닌") 12~18문항**. 부정 표현은 `<b>옳지 않은</b>`처럼 굵게.
- 전 문항 심화: 단순 정의 확인("…란 무엇인가?") 금지. 함정 비교·메커니즘·계산·헷갈리는 숫자를 적극 활용.

### 2-3. 정답 인덱스 사전 배정 (기계적으로 수행)
- 객관식 35문항: **0,1,2,3을 각 8회씩 적은 32개 수열에, 서로 다른 인덱스 3개를 1회씩 추가**해 35개를 만든다(결과: 9·9·9·8). 같은 숫자가 3연속되지 않게 섞은 뒤, 계획표 순서대로 배정한다.
- OX 5문항: O 3·X 2 또는 O 2·X 3.
- 보기를 쓸 때 처음부터 이 배정 위치에 정답을 놓는다(다 쓰고 나서 옮기지 말 것).

## 3. 문항 작성 규칙 — 공인중개사 실전 스타일

### 3-1. 발문(문두)
실제 시험 발문 형식을 그대로 쓴다:
- "…에 관한 설명으로 옳은 것은?" / "…에 관한 설명으로 <b>옳지 않은</b> 것은?"
- 조합형: "…을 모두 고른 것은?" + 진술 4개를 `<br>ㄱ. …<br>ㄴ. …<br>ㄷ. …<br>ㄹ. …` 형식으로 문제문 안에 나열. 옳은 진술 개수는 1~3개 사이에서 매일 다양하게(항상 2개 금지).
- 계산형: 모든 조건을 문제문에 명시하고 끝에 "(다른 조건은 동일)" 부가.
- 사례형: "갑은 …하였다. 이에 관한 설명으로 옳은 것은?"

### 3-2. 보기(지문) — 변별력 설계
- opts에는 내용만 쓴다(①·A 같은 번호 금지 — 렌더러가 자동으로 붙인다).
- **오답 보기는 정답과 '한 끗 차이'로 설계**: 숫자 스왑(3천만↔5천만), 주체 교체(국장↔시·도지사), 기간 교체(7일↔10일), 원칙↔예외 뒤집기, 비슷한 제도 교차(추정↔간주, 자연취락↔집단취락, 표준세율−중과기준 ↔ 중과기준만).
- 4개 보기 모두 그럴듯해야 한다. 누가 봐도 틀린 더미 보기 금지.
- 정답 보기가 유독 길거나 짧지 않게(길이로 정답 노출 금지).
- 조합형 선택지는 `["ㄱ, ㄴ","ㄴ, ㄹ","ㄷ, ㄹ","ㄱ, ㄷ"]`처럼 조합 4개.
- 문자열 내부에 큰따옴표(")와 백틱(`) 사용 금지 — 검증 스크립트가 깨진다. 따옴표가 필요하면 ' '를 쓴다.

### 3-3. 해설(expl) — 3문장 구조, 250자 이내
`[정답 근거] → [왜 그렇게 설계됐는지 메커니즘 1문장] → [함정 포인트·비교 1문장]` 순서로. OX는 반드시 "O." 또는 "X."로 시작. 계산형은 산식 전개를 포함.

### 3-4. 출처(src)
`YYYY-MM-DD 과목 소제목` 형식으로, 실제 읽은 노트의 것만 적는다. 재도전은 끝에 `(재도전)`.

## 4. HTML 생성 (기존 템플릿 복제)

- 템플릿: 데일리퀴즈 폴더에서 **파일명이 `YYYY-MM-DD.html` 표준형인 가장 최근 파일**.
- 다음만 교체하고, **CSS·렌더링·채점·결과 자동 다운로드(downloadResult) 로직은 절대 수정 금지**:
  1. `<title>`과 header `.meta` → "2026년 M월 D일 (요일) · 실전형 40문항 · 오답 재도전 N문항(🔁) 포함"
  2. `.tags` → `🔥 심화` `📝 실전 4지선다` `🔁 오답 재도전 N` `🎯 정답분산` + 오늘 다룬 과목명들
  3. `const QUIZ_DATE = "{오늘날짜}"`
  4. `QUESTIONS` 배열 → 오늘 40문항
  5. 정적 숫자 표기: 결과창 `<small> / 30</small>` → ` / 40`, 진행바 초기 문구 `0 / 30 선택` → `0 / 40 선택` (이미 40이면 그대로)
- 저장: `/Users/hg/Desktop/velog-posts/공인중개사/데일리퀴즈/{오늘날짜}.html`
