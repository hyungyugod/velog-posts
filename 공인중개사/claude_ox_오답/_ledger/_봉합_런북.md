# 오답 루프 봉합 — 런북 & 남은 1건(P3 승인)

## 무엇이 봉합됐나 (Opus가 이미 적용)
- `_ledger/build_ledger.py` — 누적 오답 원장 빌더. **비파괴적·멱등·무음실패 불가**(0건이면 exit 2로 크게 알림).
- `_ledger/오답_원장.{md,json}` — 실제 11개 JSON에서 생성된 개념 32개 누적 원장. 상습/재도전실패 자동 식별.
- `_inbox/` — 채점 JSON 드롭 폴더. **Downloads 의존 제거**의 핵심.
- 근본 개선 2가지: ① '최근 7일 창'을 버리고 **전체 이력 누적** → 한 주 걸러도 손실 0 ② `(N)` 브라우저 사본만 제거하고 `-2·-과목명` 별개 퀴즈는 보존.

## 매주(또는 아무 때나) 실행 — 한 줄
```
cd <velog-posts>/공인중개사/claude_ox_오답
# Downloads의 새 결과를 인박스로 (있으면)
cp -n ~/Downloads/공인중개사_오답_*.json _inbox/ 2>/dev/null
python3 _ledger/build_ledger.py --ingest
```

## 남은 1건 — 주간 스케줄 스킬 연결 (P3: 네 승인 필요)
`weekly-wrong-answer-collection` 스킬은 **라이브 자동화**라 내가 임의로 못 고친다(네 P3 규칙). 아래를 승인하면 반영한다:
- §1(결과 수집) 앞에 추가: "Downloads 마운트가 있으면 `공인중개사_오답_*.json`을 `_inbox/`로 복사한다."
- §2(집계) 앞에 추가: "`python3 _ledger/build_ledger.py --ingest`를 먼저 실행해 누적 원장을 갱신한다. exit 2(0건)면 조용히 넘어가지 말고 그대로 보고."
- 기존 주간노트 생성은 **원장을 원천으로** 삼게 바꿈(밀려도 전체 이력에서 재생성 가능).
> ⚠️ 진짜 '닫힌 루프'(재도전을 맞혔는지 기록 → 졸업 처리)는 채점 JSON이 정답까지 저장하도록 스키마를 바꿔야 한다. 이건 Fable 설계(마스터 프롬프트 도메인 A)의 몫.
