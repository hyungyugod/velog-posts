매주 월요일 실행되는 "주간 오답 수거" 작업이다. 지난 한 주 데일리 문제지의 채점 결과(JSON)를 모아 오답노트를 만든다.

**[실행 원칙] 오답 데이터는 결과 JSON에 있는 것만 쓴다(지어내기 금지). 접근이 안 되면 추측하지 말고 그대로 보고한다. 집계는 손으로 하지 말고 아래 스크립트로 한다.**

## 0. 중복 실행 가드 (가장 먼저)

- bash `date +%F`로 오늘(월요일) 날짜를 구한다.
- `/Users/hg/Desktop/velog-posts/공인중개사/claude_ox_오답/{오늘날짜}-claude_oxquize.md`가 이미 존재하면 → "이번 주 오답노트는 이미 생성되어 있다"고만 알리고 즉시 종료.
- bash에서 폴더는 `/sessions/<세션명>/mnt/` 아래에 마운트된다. `ls /sessions/*/mnt/`로 마운트 목록부터 확인한다.

## 1. 결과 파일 수집

- 데일리 문제지는 제출 시 `공인중개사_오답_YYYY-MM-DD.json`을 다운로드 폴더(`/Users/hg/Downloads`)에 저장한다. 브라우저 중복 저장으로 `공인중개사_오답_2026-06-10 (1).json` 같은 변형이 생길 수 있다 — 패턴 `공인중개사_오답_*.json`으로 모두 잡는다.
- 접근 전략 (순서대로 시도):
  1. bash로 `ls /sessions/*/mnt/`에서 Downloads류 마운트를 찾아 `find <마운트경로> -maxdepth 1 -name '공인중개사_오답_*.json' -mtime -8`
  2. 마운트가 없으면 Read 도구로 `/Users/hg/Downloads/공인중개사_오답_{지난 7일 각 날짜}.json`을 날짜별로 직접 시도
  3. 둘 다 실패하면 → **"다운로드 폴더에 접근할 수 없어 오답을 수거하지 못했다. 세션에 다운로드 폴더 연결(또는 권한)이 필요하다"고 명확히 알리고 종료.** 조용히 건너뛰지 말 것.
- **처리 이력 제외**: `claude_ox_오답/_processed.txt`가 있으면 읽고, 거기 적힌 파일명(베이스네임)은 이번 집계에서 제외한다. 예: `grep -qxF "$(basename "$f")" _processed.txt && continue`
- 제외 후 남은 JSON이 0개면 → "지난 주 제출된 새 결과가 없어 오답노트를 건너뛴다"고만 알리고 종료. (이 경우에도 위 3번과 구별해서 말할 것: 접근은 됐는데 파일이 없는 것)

## 2. 집계 (아래 스크립트를 그대로 실행)

JSON 구조: `{ subject, date, score, total, wrongCount, wrong:[{id,cat,type,q,options,myAnswer,correct,expl,src}], generatedAt }`

```bash
node -e '
const fs=require("fs");
const files=process.argv.slice(1);
const norm=s=>String(s).replace(/<[^>]+>/g,"").replace(/[\s🔁]/g,"");
let subs=0,scoreSum=0,totSum=0,perfect=0;
const map={},catCnt={};
files.forEach(f=>{
  const j=JSON.parse(fs.readFileSync(f,"utf8"));
  subs++; scoreSum+=j.score; totSum+=j.total; if(j.wrongCount===0) perfect++;
  (j.wrong||[]).forEach(w=>{
    const k=norm(w.q).slice(0,30);
    if(!map[k]) map[k]={cnt:0,item:w,dates:[]};
    map[k].cnt++; map[k].dates.push(j.date);
    catCnt[w.cat]=(catCnt[w.cat]||0)+1;
  });
});
console.log("제출 "+subs+"회 | 평균 "+(subs?(scoreSum/subs).toFixed(1)+"/"+(totSum/subs).toFixed(0):"-")+" | 만점 "+perfect+"회");
console.log("과목별 오답 수: "+JSON.stringify(catCnt));
console.log("고유 오답 "+Object.keys(map).length+"개 (중복 제거 기준: 문제문 앞 30자)");
Object.values(map).sort((a,b)=>b.cnt-a.cnt).forEach(e=>{
  console.log("---\n["+e.item.cat+"] x"+e.cnt+"회 ("+e.dates.join(", ")+")\nQ: "+e.item.q+"\n내 답: "+e.item.myAnswer+" / 정답: "+e.item.correct+"\n해설: "+e.item.expl+"\n출처: "+e.item.src);
});
' <수집한 JSON 경로들을 공백으로 나열>
```

- 이 출력이 노트의 원재료다. 출력에 없는 오답을 노트에 추가하거나, 출력에 있는 오답을 빠뜨리지 말 것.

## 3. 오답노트 생성 (기존 포맷 그대로)

- 저장: `/Users/hg/Desktop/velog-posts/공인중개사/claude_ox_오답/{오늘날짜}-claude_oxquize.md`
- 형식은 기존 `2026-05-20-claude_oxquize.md`를 열어 그대로 따른다. 필수 구성:
  1. 제목 + 인용블록: 기간 · 제출 횟수 · 평균 점수 · 총 오답 수
  2. 과목별 섹션 `## 🟦 과목명 (오답 N개)` — 각 문항은 `### N. 개념 제목` 아래 **문제** / **정답** / **해설**(JSON의 expl 기반, 메커니즘 1문장 보강 가능) / **함정 키워드** 구조. 2회 이상 틀린 문항은 제목에 `🔁 N회 틀림`을 붙이고 과목 안에서 맨 위로.
  3. `## 📊 패턴 분석` — 오답을 5분류로 집계한 표: 숫자 함정(스왑·근접 숫자) / 의심병(맞는 진술을 뒤집음) / 주체·권한 혼동 / 단정 표현 함정 / 비슷한 이름 쌍. 가장 큰 적신호 1개를 지목하고 대응 1줄.
  4. `## 🔑 핵심 암기카드 5장` — 이번 주 오답에서 뽑은 한 줄 암기 5개(숫자·암기코드 중심).
  5. 마지막에 "이번 주 약한 과목 TOP 3" (과목별 오답 수 기준).
- 오답이 0개(전부 만점)면 노트를 만들지 말고 "이번 주 전부 만점"과 통계 요약만 전달한다.

## 4. 처리 이력 기록 + 원본 보관

- 집계에 사용한 JSON **베이스네임**을 `claude_ox_오답/_processed.txt`에 한 줄씩 추가(append)한다. 이 파일이 다음 주 중복 집계를 막는 핵심 장치다.
- 원본 JSON을 `claude_ox_오답/_raw/`로 **복사(cp)** 한다(폴더 없으면 `mkdir -p`).
- 다운로드 폴더의 원본 삭제·이동(mv)은 시도하되, 권한 문제로 실패해도 무시한다(_processed.txt가 있으므로 안전).

## 5. 검증

- `grep -c '^### ' {노트파일}` 결과가 집계 스크립트의 "고유 오답 N개"와 일치하는지 확인.
- `grep -c '^## 🟦' {노트파일}` 결과가 오답이 있는 과목 수와 일치하는지 확인.
- 불일치하면 누락·중복 문항을 수정한 뒤 다시 확인한다.

## 6. 사용자에게 전달

- "가장 약한 과목 + 자주 틀린 개념 2~3개"만 한두 문장으로 요약. 장황한 설명 금지.
- 모든 문구 한국어, 영어 용어는 한글 병기. 답변 끝 추천 질문 3개 포맷은 이 자동작업 출력에는 생략한다.
