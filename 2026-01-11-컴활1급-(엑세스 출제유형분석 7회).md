## 1-5. visible
- 형식 탭의 표시를 아니오로 하면 특정 컨트롤이 폼이 가동되면 보이지 않도록 할 수 있고 일반적으로 이 속성을 선택하면 해당 컨트롤과 연계된 레이블 상자도 같이 표시되지 않는다.
- 개체명.visible = false로 이벤트 프로시저로 위의 작동을 이벤트 프로시저를 작성할 수 있다.

```
Private Sub Form_Load()

txt납품일자.Visible = False

End Sub
```

## 1-6. 하위폼의 정보 가져오기
- 하위폼 정보: [폼이름].[form]![컨트롤명] 이다.
- 옆에 ...을 누르면 식요소, 식범주, 식값을 통해 위의 식을 쉽게 작성할 수 있다.

## 1-7. dsum
- 함수명("계산할 필드","테이블(쿼리)","조건")
- 조건에 맞는 자료의 합을 도출한다.

> =IIf(DSum("금액","거래내역","금액>=500000 and 금액<=1000000")>=4000000,"고액구매","일반구매") 이와같이 작성하면 되고 사이라고 하면 이상, 이하 값을 포함해야 한다. 
> 이때 필드명을 []로 전달하면 안되는 이유는 DSum의 첫째/둘째 인수는 “문자열”이어야 하는데, “식별자(필드 참조)”로 넣으면 당연히 작동을 안하기 때문이다.

## 1-8. 컨트롤에 바운드되어있는 테이블을 이용하여 값 표시하기
- cmb학과명.column(1) -> 0부터 시작한다.
- cmb학과명에 바운드되어있는 테이블의 2번째 열의 값을 가져온다.

```
Private Sub Cmb학과명_BeforeUpdate(Cancel As Integer)

Txt교수번호 = Cmb학과명.Column(1)

End Sub
```

> 특정 콤보상자를 선택했을때 특정 컨트롤에 특정값이 입력되려면 콤보상자의 행원본에 특정값의 정보가 들어있어야만 한다. 따라서 행원본에서 ...을 눌러서 참조된 행원본의 열 개수를 세서 필요한 열의 인덱스를 알아내서 column으로 참조하면된다.

> iif에서는 또 =IIf([취득구분]="필기합격","*") 이렇게 한다.

## 1-11. 날짜 서식코드
- 월

| 코드   | 표시 예    | 의미               |
| ---- | ------- | ---------------- |
| m    | 1       | 월을 숫자로 표시        |
| mm   | 01      | 월을 두 자리 숫자로 표시   |
| mmm  | JAN     | 월을 영문 약자로 표시     |
| mmmm | JANUARY | 월을 영문 전체 이름으로 표시 |

- 일, 요일

| 코드   | 표시 예   | 의미                |
| ---- | ------ | ----------------- |
| d    | 1      | 일을 숫자로 표시         |
| dd   | 01     | 일을 두 자리 숫자로 표시    |
| ddd  | SUN    | 요일을 영문 약자로 표시     |
| dddd | SUNDAY | 요일을 영문 전체 이름으로 표시 |
| aaa  | 일      | 요일을 한글 약자로 표시     |
| aaaa | 일요일    | 요일을 한글 전체 이름으로 표시 |

- 오전, 오후

| 코드    | 표시 예  | 의미                        |
| ----- | ----- | ------------------------- |
| ampm  | ampm  | 오전/오후 표시 (한글 시스템이면 오전/오후) |
| am/pm | am/pm | AM / PM 형식으로 표시           |

- 또 시분초에서 분이 n임을 조심해야한다.

## 1-12. cmb코드 컨트롤의 첫번째 필드에 제품코드 필드가 지정되어있다는 의미
- 이는 제품코드 필드를 이용하라는 의미이고 필터조건에서 cmb코드.column(0)을 이용하라는 것을 의미한다.
- 또 폼이 가동되었을때 바운드되지 않고 나타나게만 하려면 폼 전체에서 작업해야한다. 이때 그냥 cmb코드 = "d1" 이런식으로 하면 된다. 이렇게 하면 바운드되지 않는다. 검색상자 자체에 컨트롤 원본을 =d1 이런식으로 안 넣었기 때문이다.

```
Private Sub cmb코드_Change()

Me.Filter = "제품코드 ='" & cmb코드.Column(0) & "'"

End Sub
```

> cmb코드.Column(0) 이 콤보상자의 값은 원본 값의 데이터타입을 따라온다.

## 1-13. 특정필드의 값을 조작하여 조건을 판단해야할때
- 특정 필드의 값들을 조작하여 조건을 판단할때는 해당 필드를 따로 빼서 조건을 위한 파트로 따로 분리하면된다.
- 또한 ringt() mod 2 이러면 right의 결과가 문자여도 mod가 숫자로 계산해주기 때문에 굳이 val을 쓸 필요가 없다.

> Expr1: Right([제품.제품코드],2) Mod 2 처럼 두개이상의 필드에 있을경우 명시적으로 어느 필드의 코드인지 명시해줘야한다.

## 1-14. cdate는 change date이다.
- cdate를 사용하여 날짜와 시간을 문자열로 합치면 날짜, 시간 필드로 잘 형식이 맞춰서 나오게 된다.
- 구체적인 형식은 해당 필드의 속성에서 정해주면 된다.
- yyyy-mm-dd hh:mm:ss 이런식으로 설정하면 된다.

> 오후 12시이후는 >=12:00 이렇게 해주면 된다.

## 1-15. tapstop과 setfocus
- tapstop과 setfocus의 차이는 tapstop은 예 아니오로 설정할 수 있는 속성이고 setfocus는 자신으로 포커스를 옮기는 메서드라는 점에서 코드 표현법이 다르다.

```
Private Sub Form_Load()

txt프로그램.SetFocus

```

## 1-16. 이벤트 프로시저에서 applyfilter
- docmd.applyfilter 필터이름, "조건" 이렇게 사용하면 되고 filteron같은 것은 따로 필요하지 않다.
- 필터이름조건은 그냥 ,로 넘어가면 된다.
- 보통이는 어딘가에 정보를 입력하고 버튼을 클릭하면 해당하는 정보를 보여준다. DoCmd.ApplyFilter는 “어떤 오브젝트에 적용할지”를 인자로 받지 않는 대신, 현재 활성화된(포커스를 가진) 오브젝트를 대상으로 동작하기때문에 가능한 것이다.

```
Private Sub cmd봉사내역_Click()

DoCmd.ApplyFilter , "학번 = '" & txt조회 & "'"

End Sub
```
