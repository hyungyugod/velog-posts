# 📌 1. 함수적 종속과 “종속자가 모든 결정자에 종속된다”의 의미

## 1-1. 함수적 종속과 결정자·종속자

- **함수적 종속(Function Dependency)**  
  - 어떤 릴레이션에서 속성 집합 $X$의 값이 정해지면 속성 집합 $Y$의 값이 유일하게 정해질 때  
    $X \rightarrow Y$ 라고 함.
- **결정자(Determinant)**  
  - 다른 속성 값을 **결정하는 속성(집합)**.  
  - 예: `학번 → 이름, 학과` 에서 `학번`이 결정자.
- **종속자(Dependent)**  
  - 결정자에 의해 **값이 결정되는 속성**.  
  - 위 예에서는 `이름`, `학과`가 종속자.

## 1-2. “모든 결정자에 종속된다” = 완전 함수적 종속

- 복합키 $(A, B)$에 대해
  - $(A, B) \rightarrow C$ 이고  
  - $A \rightarrow C$, $B \rightarrow C$ 가 **성립하지 않을 때**  
  → $C$는 $(A, B)$에 대해 **완전 함수적 종속(Full Functional Dependency)**.
- 의미:
  - 종속자가 키의 **일부가 아니라 전체**에 의해서만 결정된다.
  - 즉 “종속자가 모든 결정자에 종속된다” =  
    복합키의 **모든 구성 요소가 함께 있어야** 그 종속자를 결정할 수 있다는 뜻.
- 중요성(2NF와의 관계):
  - **제2정규형(2NF)**는 기본키의 **부분집합에만 종속(부분 함수적 종속)**되는 속성을 제거하는 정규형.
  - 따라서 “모든 결정자(복합키 전체)에 종속”될수록 2NF 조건에 가까워짐.

# 📌 2. OSI 7계층에서 전송계층의 위치와 역할

## 2-1. OSI 7계층 구조

1. 물리(Physical)  
2. 데이터링크(Data Link)  
3. 네트워크(Network)  
4. **전송(Transport)**  
5. 세션(Session)  
6. 표현(Presentation)  
7. 응용(Application)

## 2-2. 전송계층은 “중간 계층”

- 물리·데이터링크·네트워크(1~3계층)는 **하드웨어/경로 중심 하위 계층**.
- 세션·표현·응용(5~7계층)은 **사용자/응용 중심 상위 계층**.
- 전송계층(4계층)은:
  - 하위(1~3)와 상위(5~7)를 연결하는 **중간 계층**.
  - 상위 입장에서는 “하위 계층”처럼 보이지만, OSI 전체에서 통상 말하는 “하위 계층(1~3)”에는 포함되지 않음.

## 2-3. 전송계층의 핵심 기능

- **종단 간(end-to-end) 통신 제어**
  - 프로세스 대 프로세스 통신.
  - 오류 제어, 흐름 제어, 분할/재조립.
- 대표 프로토콜:
  - **TCP**: 연결 지향, 신뢰성 보장.
  - **UDP**: 비연결, 빠르지만 신뢰성 보장 없음.

# 📌 3. POP3와 교착상태(Deadlock) 필요조건

## 3-1. POP3 (Post Office Protocol v3)

- **용도**: 메일 서버에서 클라이언트로 메일을 **다운로드**하는 응용 계층 프로토콜.
- 기본 포트: TCP 110번 (보안 POP3S: 995).
- 동작 단계:
  1. **인증(Authorization)**: `USER`, `PASS`.
  2. **처리(Transaction)**: `STAT`, `LIST`, `RETR`, `DELE` 등.
  3. **갱신(Update)**: `QUIT` 시 삭제 예약 메일 실제 삭제.
- 특징:
  - 메일을 **로컬로 내려받아/offline** 사용.
  - 서버 내 메일을 삭제하는 방식이 일반적 → 다중 기기 동기화엔 불리 (IMAP과 대비).

## 3-2. 교착상태의 4가지 필요충분 조건

- 교착상태(Deadlock): 여러 프로세스가 서로가 가진 자원을 기다리며 **영원히 대기**하는 상태.

### (1) 상호배제(Mutual Exclusion)
- 자원은 동시에 **한 프로세스만** 사용할 수 있음.
- 예: 프린터 1대를 두 프로세스가 동시에 쓸 수 없음.

### (2) 점유와 대기(Hold and Wait)
- 프로세스가 **이미 자원을 점유한 상태에서**, 추가 자원을 요청하며 **대기**.

### (3) 비선점(No Preemption)
- 할당된 자원을 OS가 **강제로 빼앗을 수 없음**.
- 자원을 가진 프로세스가 스스로 반납해야 함.

### (4) 환형 대기(Circular Wait)
- 프로세스들이 자원을 **원형으로 기다리는** 관계:
  - $P_1$은 $P_2$가 가진 자원 기다림  
  - $P_2$는 $P_3$의 자원 기다림  
  - …  
  - $P_n$은 다시 $P_1$의 자원 기다림

> 위 4가지 조건이 **모두 동시에** 만족될 때 교착상태가 발생한다  
> → 이들 조건은 교착상태의 **필요충분 조건**.

# 📌 4. 시스템 인증의 종류와 인덱스 생성 SQL

## 4-1. 시스템 인증의 3가지 분류

### 4-1-1. 소유 기반 인증 (Something you have)

- **사용자가 가진 것**으로 인증.
- 예: OTP, 보안카드, 스마트카드, 휴대폰 인증서 등.

### 4-1-2. 지식 기반 인증 (Something you know)

- **사용자가 알고 있는 정보**로 인증.
- 예: 비밀번호, PIN, 보안질문.

### 4-1-3. 생체 기반 인증 (Something you are)

- **신체적 특징**을 이용.
- 예: 지문, 홍채, 얼굴 인식, 음성.

### 4-1-4. 다요소 인증 (MFA)

- 둘 이상을 결합 (예: 비밀번호 + OTP).

## 4-2. INDEX 생성 SQL 문법

```sql
CREATE INDEX index_name
ON table_name (column1, column2, ...);
```

- **UNIQUE INDEX**: 중복 불가.
- **복합 인덱스**: 여러 컬럼에 인덱스.
- **DROP INDEX index_name;** → 인덱스 삭제.

# 📌 5. DDL·DML·DCL과 하향식 통합(깊이우선·너비우선)

## 5-1. SQL 명령어 분류

| 분류 | 의미 | 주요 명령 |
|------|------|-----------|
| DDL | 데이터 구조 정의 | CREATE, ALTER, DROP |
| DML | 데이터 조작 | SELECT, INSERT, UPDATE, DELETE |
| DCL | 권한 제어 | GRANT, REVOKE |

- DDL: 자동 커밋, 롤백 불가.
- DML: 트랜잭션 제어 가능.
- DCL: 권한 관리.

## 5-2. 하향식 통합(Top-Down Integration)

- 상위 모듈부터 하위로 테스트.

### (1) 깊이우선(Depth-First)
- 한 경로를 끝까지 통합.
- 기능 단위 검증이 빠름.

### (2) 너비우선(Breadth-First)
- 같은 레벨 모듈 먼저 통합.
- 전체 흐름 조기 검증.

# 📌 6. 디스크 스케줄링, Working Set, Mesh Network, 메모리 할당 전략

## 6-1. 디스크 스케줄링 알고리즘

| 알고리즘 | 설명 | 특징 |
|-----------|------|------|
| FCFS | 도착 순서대로 | 공평하지만 느림 |
| SSTF | 가장 가까운 요청 우선 | 평균 탐색시간 ↓, 기아 발생 가능 |
| SCAN | 왕복형(엘리베이터) | 응답시간 균형 |
| C-SCAN | 한 방향만 이동 | 응답시간 일정 |
| LOOK/C-LOOK | 요청 있는 곳까지만 이동 | 효율적 |

## 6-2. Working Set

- 일정 시간 동안 실제 참조된 페이지 집합.
- $W(t, \Delta)$ 로 표현.
- 스래싱 방지, 메모리 관리 효율 향상.

## 6-3. Mesh Network

- 노드가 상호 연결된 구조.
- **Full Mesh**: 모두 연결 / **Partial Mesh**: 일부 연결.
- 장점: 신뢰성↑ / 단점: 비용, 복잡도↑.

## 6-4. 메모리 관리 전략

| 방식 | 설명 | 장점 | 단점 |
|------|------|------|------|
| First Fit | 첫 번째로 맞는 공간 | 빠름 | 단편화 발생 |
| Best Fit | 가장 작은 여유공간 | 낭비↓ | 작은 홀 증가 |
| Worst Fit | 가장 큰 공간 | 큰 작업 대비 | 낭비↑ |

# 📌 7. 교착상태 해결방안과 IPS

## 7-1. 교착상태 해결방안

| 방법 | 설명 | 장단점 |
|------|------|--------|
| 예방(Prevention) | 필요조건 중 하나 제거 | 자원낭비 가능 |
| 회피(Avoidance) | 안전상태 유지 (Banker’s) | 미리 요구량 필요 |
| 탐지(Detection) | 발생 후 탐지 | 오버헤드↑ |
| 회복(Recovery) | 탐지 후 해제 | 작업 손실 가능 |

## 7-2. IPS (Intrusion Prevention System)

- **침입 방지 시스템**  
  IDS는 탐지, IPS는 탐지+차단.
- 인라인 구조로 트래픽 차단 가능.
- 공격 시 패킷 폐기, 세션 종료, IP 차단 수행.
