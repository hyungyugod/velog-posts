# 📌 1. AVERAGE / SUM과 빈칸·오류값 처리

## 1-1. 빈칸(Empty cell)의 처리

- **SUM**  
  - 범위에 빈칸이 있으면 무시한다.  
  - 예: 1, (빈칸), 2 → =SUM(...) 결과 = 3

- **AVERAGE**  
  - 빈칸을 표본에서 제외한다.  
  - 예: 1, (빈칸), 2 → 평균 = (1+2) / 2 = 1.5

## 1-2. 오류값(#N/A 등)의 처리

- 범위에 오류가 하나라도 있으면 SUM, AVERAGE는 오류를 전파한다.

## 1-3. 오류/빈칸 처리 패턴

- SUM(IFERROR(A1:A10,0))  
- AVERAGE(IFERROR(A1:A10,""))  
- AVERAGEIF(A1:A10,"<>#N/A")

# 📌 2. COUNTIF 조건 문자열과 절대참조

## 2-1. ">="을 문자열로 쓰는 이유

COUNTIF의 조건은 작은 “문자열 언어”로 인식되기 때문.  
">=90"을 파싱하여 연산자>값으로 분리함.

## 2-2. 절대참조 빠르게 적용: F4

드래그 후 F4 누르면  
C2:C7 → $C$2:$C$7 → C$2:C$7 → $C2:$C7 순환.

# 📌 3. SUMIF 절대·상대참조

=SUMIF($B$2:$B$7,"=한국고교",C2:C7)  
조건범위는 고정, 합계범위는 이동 가능. 정상 구조.

# 📌 4. 금융 함수 PV / FV / PMT

| 항목 | 의미 | 기능 | 상황 |

| PV | 현재가치 | 미래 돈→현재 기준 | 대출 원금 |

# 📌 5. 배열 수식 vs 일반 수식

## 5-1. 일반 수식

한 값만 계산.

=SUM(A1:A5 * B1:B5)  
과거엔 반드시 CSE 필요.

# 📌 6. 데이터 표의 행·열 입력 셀

데이터 표는 “원본 수식 하나”를 여러 조합으로 반복 계산하여 결과 매트릭스를 만드는 자동 실험 장치.

- 행 입력 셀: 행 방향 변수  
- 열 입력 셀: 열 방향 변수  

즉, 데이터 표 숫자를 직접 계산하지 않고 **원본 수식을 계속 돌려 값 채우기** 때문에 반드시 지정해야 함.

# 📌 7. VBA에서 세미콜론이 아닌 콜론을 쓰는 이유

BASIC 계열은 줄바꿈이 문장 종료.  
한 줄에 여러 문장을 넣을 때만 콜론(:) 사용.  
세미콜론은 출력 제어에 이미 사용 중.

# 📌 8. 매개변수 쿼리 vs 크로스탭 쿼리

## 8-1. 매개변수 쿼리  
실행 시 사용자 입력값을 조건으로 쓰는 쿼리.

## 8-2. 크로스탭 쿼리  
행·열 기준에 따라 요약표(피벗형)를 만드는 쿼리.

# 📌 9. 불일치 검색 쿼리·필드 순서·필드명과 테이블명

## 9-1. 불일치 검색이 테이블 1개에서 불가능한 이유  
A에는 있지만 B에는 없는 값을 찾는 구조라 비교 대상 2개 필요.

## 9-2. 필드 순서를 잘라내기/붙여넣기로 옮기면 안 되는 이유  
필드 스키마가 UI가 아닌 테이블 구조 수준에서 변경되기 때문.  
데이터 자체 이동 위험.  
디자인 보기에서 드래그로 바꿔야 함.

## 9-3. 필드 이름과 테이블 이름이 같아도 되는 이유  
이름 공간(namespace)이 다르다.  
SELECT 고객 FROM 고객; → 앞은 필드, 뒤는 테이블.

# 📌 10. 필드명이 숫자로 시작해도 되는 이유

필드명은 식별자가 아니라 레이블.  
[...] 로 감싸면 문자열 이름이므로 숫자 시작 가능.

반면 테이블명·변수명은 언어 식별자 규칙을 따라 숫자 시작 불가.

# 📌 11. XLOOKUP과 머리글/바닥글 코드 &[...]

## 11-1. XLOOKUP 핵심  
방향 제한 없고 정확일치 기본이며 못 찾을 때 기본값 지정 가능.

## 11-2. 머리글/바닥글 코드

&[Page], &[Pages], &[File], &[Tab] 등.  
한글 코드(&[페이지])는 단순 텍스트.  
영문 코드만 기능 수행.

# 📌 12. R1C1 참조 스타일

## 12-1. A1  
열=문자, 행=숫자.

## 12-2. R1C1  
행·열 모두 숫자.  
상대참조 표현 강력: R[0]C[-1] 등.  
VBA 내부는 기본적으로 R1C1 사용.

# 📌 16. 배열 상수와 오류값

엑셀은 오류값도 하나의 “값” 타입으로 취급.  
배열은 혼합 타입 허용.

# 📌 17. BIOS/COSMO 비밀번호와 배터리 제거

- 비밀번호는 CMOS(휘발성)에 저장.  
- CMOS는 메인보드 배터리로 유지.  
- 배터리 제거 → CMOS 초기화 → BIOS 설정·비밀번호 삭제.  

따라서 비밀번호를 잊었을 때 해결책이 될 수 있음.

# 📌 18. CMD 삭제가 휴지통을 거치지 않는 이유

휴지통은 Explorer.exe가 구현한 “특수 이동” 기능.  
CMD의 del/erase/rd는 파일 시스템 API를 직접 호출하여 즉시 삭제.  
NTFS에서는 실제 데이터는 바로 지워지지 않고 MFT에서 “사용 가능” 표시만 바뀜.

# 📌 19. 인터프리터 언어: 수정 쉬움 / 소스 공개

- 수정 쉬움: 컴파일 없이 소스를 즉시 실행.  
- 소스 공개: 배포 형태가 사람이 읽을 수 있는 코드 그대로.  
- 보안 취약 → 난독화/패키징 등으로 보완.

# 📌 20. 반송파(carrier) 발생

- 원본 데이터(음성·전압 신호)는 저주파라 직접 전송 불리.  
- 멀리 전파될 수 있는 고주파 사인파(반송파)를 먼저 생성.  
- 데이터 모양대로 반송파의 진폭·주파수·위상을 변조하여 태움.  
- 수신 측은 반송파에서 데이터만 복조.

비유: 데이터=손님, 반송파=기차, 변조=승차, 복조=하차.
