## 1-3. 한 테이블의 두 필드를 가지고 유효성 검사하기
- 한 테이블에 두가지 필드를 사용하려면 필드 속성에서는 할 수 없고 테이블 속성 즉 테이블 디자인의 속성시트를 클릭해서 나오는 테이블 속성에서 전체 유효성 검사 규칙을 설정할 수 있다.
- 이떼 필드명은 큰괄호로 감싸서 필드명이라는 것을 알려주어야 한다.

## 1-4. 조회속성에서 특정필드를 표시하지 않기, 열 이름 표시하기
- 조회속성에서 특정필드를 표시하지 않기 위해서는 열 너비를 0으로 주면 된다. 그리고 열 이름 즉 필드명을 조회 목록에 표시하기 위해서는 열 이름 속성을 예로 주면 된다.
- 추가적으로 열 개수는 사진을 보고 열개수를 정해줘야 한다. 또 이때 숨겨지는 열도 카운트 해주어야한다.
- 필드에 제품코드를 저장하려면 일단 select는 해와야 한다.

## 1-8. 쿼리를 실행시키는 runsql
- 우선 등록생신을 눌러서 테이블을 업데이트하고 새로고침하여 폼에 이를 반영한다.
- docmd.runsql "sql명령" 으로 sql문을 실행시킬 수 있다.
- 이후 me.requery나 docmd.requery를 통해 한번 더 쿼리를 작동시켜 폼을 새로고침할 수 있다.
- 또 이런 실행쿼리들은 한번 실행하여 제출하는 것이 맞다.

```
Private Sub Cmd등록갱신_Click()

DoCmd.RunSQL "update 등록자료 set 보고자수 = 보고자수 * 1.1"
Me.Requery

End Sub
```

## 1-11. 보고서에서 모든 레코드에 적용하는 조건부 서식
- 한 줄을 ctrl로 선택하고 보고서의 서식-조건부서식으로 조건부 서식을 만든다.
- 엑셀과 달리 =으로 시작하지 않는다. 
- 필드명에 큰괄호를 꼭 넣어줘야 한다.
- 이름 like "*이*" 이렇게 주면 된다.
- instr > 0으로 적용해도 된다.

## 1-12. 엑세스의 mod
- 엑세스에서 mod는 2mod3이런식으로 2를 3으로 나눈 나머지를 구한다.

> =IIf([Page] Mod 2=1,[Page] & "페이지") 인데 만약 짝수부는 어떻게 하란 말이 없으면 그냥 비워서 조건을 설정하지 않으면된다.

## 1-13. recordsource 속성
- recordsource 속성은 레코드 원본 속성을 의미한다. 여기에 원본을 입력하면 해당 테이블의 속성들을 가져와 사용할 수 있게 된다.
- me.recordsorce = "sql문" 이렇게 사용해야한다.
- 보통 레코드 원본을 지정할때는 필드명에 *을 사용하여 전체 필드를 가져온다.
- 이때 cmb조회 같은 외부값을 가져올때 문자열로 묶여있으면 안되므로 필터에서 처럼 &로 연결해서 where절을 잘 작성해주어야한다.

```
Private Sub cmb조회_Click()

Me.RecordSource = "select * from 요금명 where 요금코드 ='" & cmb조회 & "'"

End Sub
```

## 1-14. having과 where
- having 조건은 평균으로 묶은 뒤에 아래에 조건을 입력하면 된다.
- 집계함수가 사용되었을때 조건을 따로 필드를 만들지 아니면 집계필드에서 조건까지 설정할지 잘 고민해야한다.

## 1-15. 하나의 테이블로 크로스탭쿼리
- 하나의 테이블로 크로스탭 쿼리를 만들때는 마법사를 사용하는 것이 좋다.

> 마법사로 크로스탭 쿼리를 만들었을때 열머리글이 전부 표시되는 문제가 발생할 수 있는데 이럴때는 그냥 속성에 들어가서 열 머리글을 전부 삭제해보면 된다.
