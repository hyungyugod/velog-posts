# 📌 1. Windows UI와 Aero

## 1-1. Aero의 개념

- **Aero**는 약자가 아니라 Windows Vista / 7에서 사용된 **UI 디자인 브랜드 이름**이다.
- 흔히 **Aero Glass Interface**라고 부르며, 창 테두리와 인터페이스를 유리처럼 **반투명·반사·블러 효과**가 나게 표현하는 디자인이다.
- 목표는 단순 기능뿐 아니라 **시각적 깊이감과 미래적인 느낌**을 주는 것이었다.

## 1-2. Aero의 주요 요소

- **Aero Shake**
  - 한 창의 타이틀 바를 마우스로 흔들면 그 외의 창을 모두 최소화한다.
  - 여러 창이 어지럽게 열려 있을 때 **집중하고 싶은 창 하나만 남기는 기능**이다.
- **Aero Snap**
  - 창을 화면의 좌우/상단 가장자리로 드래그하면 자동으로 절반 화면, 최대화 등으로 정렬한다.
  - 멀티태스킹과 창 정리에 유용하다.
- **Aero Peek**
  - 작업 표시줄 오른쪽 끝에 마우스를 올리면 모든 창이 투명해지며 바탕화면을 잠시 확인할 수 있다.
- **Aero Glass (투명/블러 효과)**
  - 창 테두리와 UI 일부가 유리처럼 반투명/블러 처리되어 **레이어 구조와 깊이감**을 준다.

# 📌 2. 코덱(codec)의 개념

## 2-1. 어원과 정의

- **Codec = COder + DECoder**
- 영상·음성 데이터 등을 **압축하여 저장/전송하고, 다시 복원하는 기술 또는 소프트웨어**를 의미한다.
- 용량이 큰 원본 데이터를 그대로 저장/전송하는 것은 비효율적이므로, 코덱으로 압축해 사용한다.

## 2-2. 왜 코덱이 필요한가?

- **용량 절감**
  - 고해상도 영상이나 고음질 음원은 원본 상태에서 용량이 매우 크다.
  - 코덱으로 압축하면 저장공간과 네트워크 대역폭을 절약할 수 있다.
- **스트리밍 가능**
  - 유튜브, 넷플릭스, 음악 스트리밍 등은 코덱을 이용해 **실시간 전송 + 디코딩**을 수행한다.
- **호환성 확보**
  - 정해진 규칙(코덱)을 이용하여 인코딩/디코딩해야 다양한 기기와 프로그램에서 동일 데이터를 재생할 수 있다.

## 2-3. 컨테이너와 코덱의 구분

- **코덱**: 데이터를 압축/복원하는 **방식** (예: H.264, H.265, AAC, MP3 등)
- **컨테이너**: 여러 스트림(영상, 음성, 자막 등)을 **하나의 파일 포맷으로 싸는 껍데기** (예: MP4, MKV, AVI 등)
- 같은 `.mp4` 파일이라도 **내부에 어떤 코덱이 들어 있는지**는 다를 수 있다.

# 📌 3. 파일 시스템과 NTFS, 플로피 디스크

## 3-1. NTFS의 정의와 특징

- **NTFS (New Technology File System)**: Windows 계열에서 사용하는 대표적인 파일 시스템.
- 역할: 디스크 위에 저장된 파일이 **어디에 위치하고, 누가 접근할 수 있으며, 에러가 났을 때 어떻게 복구할지**를 관리한다.

### NTFS의 주요 기능

- **권한 관리(ACL)**: 사용자/그룹별 읽기, 쓰기, 실행 권한 설정.
- **저널링(Journaling)**: 쓰기 작업을 로그에 남겨, 정전이나 시스템 오류 후에도 파일 시스템의 일관성을 회복하기 쉽게 한다.
- **대용량 지원**: 매우 큰 파티션과 파일 크기 지원.
- **암호화(EFS), 압축, 링크(하드 링크, 심볼릭 링크)** 등 고급 기능 제공.

## 3-2. 플로피 디스크와 NTFS의 관계

- **플로피 디스크**: 과거 사용하던 휴대용 저장매체로, 용량은 보통 **1.44MB** 정도에 불과하다.
- 플로피 디스크는 **FAT12** 같은 매우 단순하고 작은 파일 시스템을 사용했다.
- **NTFS는 구조가 복잡하고 메타데이터가 많아 플로피처럼 극소용량 매체에는 적합하지 않다.**
  - 그래서 "NTFS가 플로피 디스크를 포맷할 수 없다"는 말은,
  - 플로피 같은 구형·저용량 매체에는 NTFS 파일 시스템을 적용할 수 없다는 뜻이다.

# 📌 4. Windows 부팅 과정: POST와 MBR

## 4-1. POST (Power-On Self Test)와 메모리 검사

- 컴퓨터 전원을 켰을 때, 가장 먼저 **BIOS/UEFI**가 실행된다.
- 이때 하드웨어(메모리, CPU, 그래픽카드, 키보드 등)의 기본 상태를 점검하는 과정이 **POST**이다.
- **POST 메모리 검사**: RAM이 정상적으로 인식되고 오류가 없는지 확인하는 단계.
  - 운영체제를 메모리에 올려야 부팅할 수 있으므로, 메모리 이상 여부는 필수적으로 점검된다.
  - 심각한 이상이 있으면 비프음으로 오류를 알리며 부팅이 중단될 수 있다.

## 4-2. MBR (Master Boot Record)의 역할

- POST가 끝난 후, BIOS/UEFI는 부팅 가능한 디스크를 찾는다.
- 디스크의 **가장 앞 512바이트** 영역에 있는 것이 **MBR**이다.
- MBR에는 크게 두 가지 정보가 들어 있다.
  - 간단한 **부트스트랩 코드**: 운영체제 로더를 불러오는 작은 프로그램.
  - **파티션 테이블 정보**: 디스크가 어떻게 나뉘어 있고, 어디에 OS가 설치되어 있는지.
- MBR이 손상되면 부팅 시 **운영체제를 찾지 못해 부팅 실패**가 발생할 수 있다.
- 최근에는 MBR 대신 **GPT**를 사용하는 UEFI 기반 구조도 많이 쓰인다.

# 📌 5. 선점형 멀티태스킹과 Windows의 DBMS 비포함 이유

## 5-1. 선점형 멀티태스킹 (Preemptive Multitasking)

- 멀티태스킹: 여러 프로그램이 **동시에 실행되는 것처럼 보이도록** CPU를 나누어 쓰는 방식.
- **선점형**이란, 운영체제가 **강제로 CPU 사용권을 회수하여** 다른 프로세스에게 배분하는 방식을 말한다.
- 특징:
  - 한 프로그램이 문제를 일으키거나 무한 루프에 빠져도, OS가 CPU를 회수해 **전체 시스템이 멈추는 것을 방지**한다.
  - 사용자가 여러 프로그램(브라우저, 메신저, 음악 등)을 동시에 사용하는 환경을 지원한다.
- 이는 **CPU 스케줄러**가 일정 시간 단위(타임 슬라이스)로 작업을 번갈아 처리하기 때문에 가능하다.

## 5-2. Windows가 데이터베이스 관리 기능(DBMS)을 포함하지 않는 이유

- 운영체제(OS)와 데이터베이스 관리 시스템(DBMS)의 역할은 다르다.
  - OS: 하드웨어 자원(메모리, CPU, 파일 시스템)을 관리하고 프로그램이 돌아갈 기반을 제공.
  - DBMS: 대규모 데이터를 저장·검색·수정하고, 트랜잭션, 동시성 제어, 복구 등의 고급 기능을 제공.
- Windows가 DBMS를 기본 포함하지 않는 이유:
  - **역할 분리**: OS는 기반 플랫폼, DBMS는 선택적으로 설치하는 응용 소프트웨어.
  - **다양한 요구**: 사용자와 환경에 따라 필요한 데이터베이스 종류와 스펙이 다르므로, 특정 DBMS를 OS에 강제 탑재하기 어렵다.
  - **선택권과 확장성**: 사용자가 MySQL, PostgreSQL, Oracle, SQL Server 등 다양한 DBMS 중에서 선택할 수 있도록 하기 위함.

# 📌 6. 수정 키(Ctrl, Shift, Alt, Esc 등)의 철학과 단축키 구조

## 6-1. 각 키의 기본 철학

- **Ctrl (Control)**
  - 의미: 제어, 명령 실행.
  - 역할: 복사, 붙여넣기, 저장, 전체 선택 등 **행동을 직접 트리거**하는 명령 중심 키.
- **Shift**
  - 의미: 변형, 이동, 확장.
  - 역할: 연속 선택, 대문자 입력, 기존 기능을 **강화·확장**하는 키.
- **Alt (Alternate)**
  - 의미: 대체, 우회.
  - 역할: 메뉴 접근, 다른 창 전환, 종료 등 **표준 흐름이 아닌 대체 경로**를 여는 키.
- **Esc (Escape)**
  - 의미: 탈출, 취소.
  - 역할: 현재 작업/모드에서 빠져나오는 **비상구 역할**.

요약하면  
$Ctrl =$ 실행, $Shift =$ 변형/확장, $Alt =$ 대체 경로, $Esc =$ 탈출/취소.

## 6-2. 대표적인 단축키 개념 정리

### 6-2-1. Ctrl 계열

- **Ctrl + C / V / X / A / Z**
  - 복사, 붙여넣기, 잘라내기, 전체 선택, 실행 취소.
  - 모두 사용자의 **행동을 직접 실행하는 제어 명령**이다.
- **Ctrl + N**
  - 새 창 / 새 문서 생성.
  - 새로운 작업 흐름을 **시작하는 제어 명령**이다.
- **Ctrl + E**
  - 파일 탐색기/브라우저 등에서 검색창 또는 주소창 포커스.
  - 현재 작업의 초점을 **탐색/검색 행동 영역**으로 옮기는 명령.

### 6-2-2. Shift 계열

- **Shift + 클릭**
  - 첫 항목과 마지막 항목 사이를 **연속 선택**.
- **Ctrl + 클릭**
  - 떨어져 있는 항목들을 **비연속 선택/해제** (토글 방식).
- **Shift + Delete**
  - 휴지통 없이 바로 삭제.
  - Delete의 동작을 **강화·위험도 상승**시키는 변형이다.
- **Shift + F10**
  - 마우스 오른쪽 클릭과 같은 컨텍스트 메뉴 열기.
  - 마우스 조작을 키보드로 **대체**한다.

### 6-2-3. Alt 계열

- **Alt + F4**
  - 현재 창/프로그램 종료.
  - 마우스로 닫기 버튼을 누르는 것의 **대체 경로**이다.
- **Alt + Tab**
  - 실행 중인 프로그램 간 전환.
  - 전환 목록을 시각적으로 보여주며, 사용자가 **보고 선택**할 수 있다.
- **Alt + Esc**
  - 별도의 목록 UI 없이, 실행 중인 창들을 **순서대로 뒤로 보내며 순환 전환**한다.
  - Alt+Tab은 "보고 고르는 전환", Alt+Esc는 "보지 않고 밀어 넘기는 전환"에 해당한다.
- **Alt + Enter**
  - 선택한 개체의 **속성 창** 열기(파일 속성, 일부 프로그램에서는 전체화면 토글 등).
  - 기본 실행이 아니라 **대체 정보/설정 레벨 접근**이라는 의미를 갖는다.

### 6-2-4. Esc와 조합

- **Esc**
  - 현재 대화상자나 메뉴, 전체화면 상태 등을 **취소/종료**.
- **Ctrl + Shift + Esc**
  - 작업 관리자 실행.
  - $Ctrl =$ 제어, $Shift =$ 강화, $Esc =$ 탈출 이 결합되어,
  - 시스템 문제 시 **가장 빠르게 관리창(작업 관리자)에 진입하는 긴급 제어 단축키**로 해석할 수 있다.

### 6-2-5. Windows 키 계열

- **Windows + D**
  - 모든 창을 최소화하고 **바탕화면 표시/복귀**.
- **Windows + E**
  - 파일 탐색기 실행 → OS 파일 계층 구조에 **즉시 접근**.
- **Windows + M**
  - 열려 있는 창 모두 최소화.
- **Windows + R**
  - 실행 창 표시 → 명령어 기반 실행의 **입구 역할**.
- **Windows + T**
  - 작업 표시줄에 있는 프로그램을 순서대로 선택.
- **Windows + Alt + D**
  - 작업표시줄 우측의 날짜/시간 영역을 표시/숨김.
- **Windows + Pause Break**
  - 시스템 속성(사양) 창 열기.

# 📌 7. 파일 드래그 동작과 Ctrl / Shift의 의미

## 7-1. 드래그 시 조합별 동작 차이

Windows에서 파일을 **다른 위치로 드래그**할 때, 누르고 있는 키에 따라 동작이 달라진다.

- **그냥 드래그**
  - 같은 드라이브 내 이동: 기본적으로 **이동(Move)**.
  - 다른 드라이브로 이동: 기본적으로 **복사(Copy)**.
- **Ctrl + 드래그**
  - 강제로 **복사(Copy)**.
  - 원본은 그대로 두고, 대상 위치에 사본 생성.
- **Shift + 드래그**
  - 강제로 **이동(Move)**.
  - 원래 위치에서 제거하고 대상 위치로 옮긴다.
- **Ctrl + Shift + 드래그**
  - 파일을 이동/복사하는 대신 **바로가기(Shortcut)를 생성**.
  - 원본은 원래 위치에 그대로 있고, 대상 위치에는 **링크(지름길)**만 생긴다.

## 7-2. 설계 철학적 해석

- $Ctrl =$ 안전한 방향(복사) 중심의 제어.
- $Shift =$ 기본 동작을 변화/강화(이동, 즉 삭제에 가까운 효과).
- $Ctrl + Shift =$ 복사와 이동 사이의 **중간 개념인 "참조 연결(바로가기)"**로 귀결.

# 📌 8. 대화상자(Dialog Box)의 개념

## 8-1. 대화상자란?

- 프로그램이 사용자에게 **질문을 하거나, 선택/입력을 요구하거나, 알림을 보여줄 때** 나타나는 **별도의 작은 창**이다.
- 예: "저장하시겠습니까?", "오류가 발생했습니다", "이름을 입력하세요" 등의 팝업 창.

## 8-2. 영어 표현

- 일반적인 명칭: **Dialog Box** 또는 **Dialogue Box**.
- 넓은 의미: **Popup Window** (모든 팝업 창을 통칭할 때).
- 현재 창을 닫기 전까지 다른 작업이 불가능한 경우: **Modal Window**.
- 단순 알림 위주라면: **Message Box**.

# 📌 9. 프린터 오류와 인쇄 작업, 스풀링

## 9-1. 프린터가 오프라인/용지 걸림 상태일 때의 의미

- "지정된 프린터가 오프라인 상태이거나 용지 걸림 상태이면, 오류가 발생한 문서를 새 포트로 보낼 수 없다"는 말의 의미:
  - 인쇄 작업은 **특정 프린터와 포트에 연결된 대기열(스풀러)**에 올라간다.
  - 해당 프린터가 오프라인이거나 에러 상태이면, 그 대기열에 묶여 있는 문서를 **다른 포트/프린터로 자동 전환하여 보내기는 어렵다**는 뜻이다.
  - 보통은 사용자가:
    1. 오류를 해결하고 프린터를 다시 온라인으로 만들거나,
    2. 해당 작업을 취소하고 **다른 프린터를 선택하여 다시 인쇄 명령**을 내려야 한다.

## 9-2. 스풀링(Spooling)의 개념

- **Spooling (Simultaneous Peripheral Operations On-Line)**:
  - 인쇄와 같이 느린 입출력 장치에 작업을 보낼 때,
  - 요청을 즉시 장치에 보내는 대신 **디스크나 메모리의 대기열(스풀러)에 저장해 두었다가 순서대로 처리하는 방식**이다.
- 필요성:
  - 프린터는 한 장씩 천천히 출력하지만, CPU는 매우 빠르다.
  - 스풀링이 없으면 CPU가 프린터가 끝날 때까지 기다려야 하므로 **전체 시스템 효율이 낮아진다**.
  - 스풀러가 인쇄 요청을 대신 받아 저장하고, 프린터는 시간이 날 때마다 하나씩 처리한다.
- 장점:
  - 사용자는 인쇄 명령 후 **바로 다른 작업을 계속할 수 있다**.
  - 여러 사용자의 인쇄 요청을 **순서대로 관리**할 수 있다.
  - 프린터가 오류 상태(용지 걸림 등)가 되어도, 작업이 스풀러에 남아 있어 **오류 해결 후 다시 출력**이 가능하다.

# 📌 10. 전체 개념 흐름 요약

## 10-1. UI와 사용자 상호작용

- Aero는 시각적 효과(투명/블러 등)를 통해 UI의 층과 깊이를 표현한다.
- 대화상자(Dialog Box)는 프로그램과 사용자가 **질문-응답 형태**로 상호작용하게 하는 창이다.
- 단축키와 수정 키(Ctrl, Shift, Alt, Esc, Win 등)는:
  - $Ctrl$: 명령 실행
  - $Shift$: 변형·확장
  - $Alt$: 대체 경로
  - $Esc$: 탈출/취소
  - $Win$: OS 레벨 기능 접근
  로 역할이 나뉘어 있으며, 이 철학에 따라 다양한 단축키가 파생된다.

## 10-2. 시스템 내부 구조와 자원 관리

- NTFS는 현대 Windows의 기본 파일 시스템으로, **권한·저널링·대용량·암호화** 등을 지원한다.
- 플로피 디스크처럼 극소용량 매체는 NTFS의 복잡한 구조를 담을 수 없어, FAT12 같은 간단한 파일 시스템을 사용한다.
- 부팅 과정에서는:
  - POST로 하드웨어(특히 메모리)를 점검하고,
  - MBR(또는 GPT)을 읽어 OS가 어디 있는지 확인한 후 부트 로더를 실행한다.
- Windows의 선점형 멀티태스킹은 OS가 CPU 사용권을 강제로 회수하며 여러 프로세스를 공정하게 실행하게 한다.
- Windows는 OS로서 **기반 플랫폼** 역할을 하고, DBMS는 선택적으로 설치하는 **별도의 소프트웨어 계층**으로 분리되어 있다.

## 10-3. 입출력 장치와 스풀링

- 인쇄 작업은 **스풀링**을 통해 스풀러(대기열)에 먼저 저장되고, 프린터는 자신의 속도에 맞추어 순차 처리한다.
- 프린터가 오프라인/에러 상태면, 그 프린터 대기열에 묶인 작업을 **바로 다른 포트/프린터로 옮기기 어렵다**.
- 일반적인 해결 방법은:
  - 오류 해결 후 다시 출력,
  - 또는 대기열에서 문서를 취소하고 **다른 프린터를 선택해 새로 인쇄 명령을 내리는 것**이다.
