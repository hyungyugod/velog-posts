# 📌 1. 엑세스 출제유형분석 7회
## 1-1. 특정 문자가 입력되지 않도록 유효성 검사규칙 설정
- 특정 문자가 입력되지 않도록 유효성 검사규칙 설정하는 법은 instr=0으로 유효성 검사 규칙을 설정해주면 된다.

## 1-2. 인덱스 설정
- 테이블 디자인 - 인덱스에서 인덱스의 고유 여부를 변경할 수 있다.

## 1-3. >
- 한번 >를 넣으면 뒤의 모든 문자는 대문자이다. 만약 중간에 하나가 소문자여야한다면 그때 <를 적어주면 된다.

## 1-4. 거래처에 목록에 없는데 거래내역에 있을 수 없다.
- 관계를 설정할때 참조무결성을 지켜주기 위해서는 부모쿼리에 없는 자식의 필드는 없애줘야한다.

## 1-5. visible
- 형식 탭의 표시를 아니오로 하면 특정 컨트롤이 폼이 가동되면 보이지 않도록 할 수 있고 일반적으로 이 속성을 선택하면 해당 컨트롤과 연계된 레이블 상자도 같이 표시되지 않는다.
- 개체명.visible = false로 이벤트 프로시저로 위의 작동을 이벤트 프로시저를 작성할 수 있다.

## 1-6. 하위폼의 정보 가져오기
- 하위폼 정보: [폼이름].[form]![컨트롤명] 이다.
- 옆에 ...을 누르면 식요소, 식범주, 식값을 통해 위의 식을 쉽게 작성할 수 있다.

## 1-7. dsum
- 함수명("계산할 필드","테이블(쿼리)","조건")
- 조건에 맞는 자료의 합을 도출한다.

## 1-8. 컨트롤에 바운드되어있는 테이블을 이용하여 값 표시하기
- cmb학과명.column(1) -> 0부터 시작한다.
- cmb학과명에 바운드되어있는 테이블의 2번째 열의 값을 가져온다.

## 1-9. setfocus도 개체의 속성이다.
- 나를 가리켜줘! 하는 느낌이다. 
- fontsize도 개체의 속성이다. visible도 이와 같다.

## 1-10. 매페이지마다 반복적으로 표시/인쇄되도록하라
- 특정 컨트롤들을 페이지 머리글로 옮기라는 뜻이다.
- 옮길때 특정 컨트롤을 빠뜨리지 않도록 주의하여야한다.
- 영역을 움직여보면서 남은 컨트롤이 있는지 확인해보면 좋다.

## 1-11. 날짜 서식코드
- 월

| 코드   | 표시 예    | 의미               |
| ---- | ------- | ---------------- |
| m    | 1       | 월을 숫자로 표시        |
| mm   | 01      | 월을 두 자리 숫자로 표시   |
| mmm  | JAN     | 월을 영문 약자로 표시     |
| mmmm | JANUARY | 월을 영문 전체 이름으로 표시 |

- 일, 요일

| 코드   | 표시 예   | 의미                |
| ---- | ------ | ----------------- |
| d    | 1      | 일을 숫자로 표시         |
| dd   | 01     | 일을 두 자리 숫자로 표시    |
| ddd  | SUN    | 요일을 영문 약자로 표시     |
| dddd | SUNDAY | 요일을 영문 전체 이름으로 표시 |
| aaa  | 일      | 요일을 한글 약자로 표시     |
| aaaa | 일요일    | 요일을 한글 전체 이름으로 표시 |

- 오전, 오후

| 코드    | 표시 예  | 의미                        |
| ----- | ----- | ------------------------- |
| ampm  | ampm  | 오전/오후 표시 (한글 시스템이면 오전/오후) |
| am/pm | am/pm | AM / PM 형식으로 표시           |

- 또 시분초에서 분이 n임을 조심해야한다.
