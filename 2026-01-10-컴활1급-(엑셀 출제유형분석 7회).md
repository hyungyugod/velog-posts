## 1-6. right, left는 반환값이 문자열이다.
- right, left는 반환값이 문자열이다. 하여 그 이상의 처리를 위해서는 value함수를 사용해주어야한다.
- hlookup은 테이블을 넣고 거기서 알아서 일치하는 찾을 값에서 원하는 행에 있는 값을 찾아준다는 점에서 2번째 인수에 시원하게 데이터를 다 넣어 주어야한다. 
- 그리고 서식으로 글자가 참조표에 붙어있어야 문제가 성립되므로 일부러 그렇게 문제를 출제한다.
- 그리고 범위는 1, 5, 7 이런식으로만 인식해서 true 유형으로 러프하게 찾는 것이다.

> =HLOOKUP(LEFT(M4,1),$N$18:$P$19,2,FALSE)&"("&HLOOKUP(VALUE(RIGHT(M4,1)),$S$18:$U$20,3,TRUE)&")" 에서처럼 숫자로 비교하게되는 RIGHT(M4,1)는 VALUE(RIGHT(M4,1))이렇게 표기해줘야 완전히 숫자인수로 대입한 꼴이 된다. 아니면 "1"이런식으로 입력하여 자동으로 처리가 안된다.

## 1-7. select case
- select case 개체명
- case "문자" 혹은 is 일반논리식
- case else
- end case

```
Public Function fn배달금액(수량, 단가, 배달방법)

Select Case 배달방법
    Case "택배"
        Select Case 수량 * 단가
            Case Is >= 30000
                fn배달금액 = 0
            Case Else
                fn배달금액 = 3000
        End Select
    Case "퀵"
        fn배달금액 = 12000
    Case "직접수령"
        fn배달금액 = 0
        
End Select

End Function
```
