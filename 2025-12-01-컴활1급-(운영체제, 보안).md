# 📌 1. 운영체제 관련 개념

## 1-1. 다중 프로그래밍(multiprogramming)과 시분할(time-sharing)

### (1) 다중 프로그래밍

- **목적**
  - CPU가 쉬는 시간을 최소화하여 **CPU 이용률을 극대화**하는 것.
- **동작 개념**
  - 여러 프로그램을 메모리에 동시에 적재.
  - 하나의 프로그램이 I/O 작업 때문에 대기 상태가 되면,  
    CPU를 다른 프로그램에게 즉시 넘겨서 계속 일을 시킨다.
- **특징 요약**
  - 기준: **CPU 입장**에서 효율을 높이는 구조.
  - 사용자 입장에서는 “여러 작업이 동시에 돌아가는 느낌”이지만  
    **응답 속도(대화형성)**이 목표는 아니다.
  - 주로 일괄 처리(batch) 환경, 서버 처리량(throughput) 향상이 중요할 때 유리.

---

### (2) 시분할 처리 시스템(Time-sharing System)

- **목적**
  - 여러 사용자가 동시에 사용하는 것처럼 보이도록 **빠른 응답성** 제공.
- **동작 개념**
  - CPU 시간을 아주 잘게 쪼갠 **타임 슬라이스(time quantum)** 단위로 사용자/프로세스에 배분.
  - A 사용자 프로그램 → B → C → 다시 A … 이런 식으로 매우 빠르게 순환.
- **특징 요약**
  - 기준: **사용자 입장**에서의 즉각적인 응답(인터랙티브 환경).
  - 터미널·콘솔 등 다중 사용자 환경에서 각각이 “나만 쓰는 것 같은 느낌”을 받도록 설계.
  - 현대 OS의 멀티태스킹 환경은 다중 프로그래밍 + 시분할을 함께 활용.

---

## 1-2. 다중처리(multiprocessing)와 분산처리(distributed processing)

### (1) 다중처리 시스템(Multiprocessing System)

- **구조**
  - 하나의 **컴퓨터 시스템 내부에 CPU(프로세서)가 여러 개** 존재.
  - 하나의 운영체제(OS)가 여러 CPU를 관리.
  - **메모리·스토리지 등 자원을 공유**하는 형태(SMP 구조 등).
- **특징**
  - 통신 속도: 메모리 버스/공유 메모리 기반 → 매우 빠름.
  - 목적: **병렬 처리·성능 향상**.
  - 예: 멀티코어 CPU, 고성능 서버, L3/L4 스위치 내부 처리 등.
- **비유**
  - “한 집(한 OS, 한 메모리)에 똑똑한 두뇌 여러 개가 같이 일하는 팀.”

---

### (2) 분산처리 시스템(Distributed System)

- **구조**
  - **물리적으로 떨어진 여러 대의 컴퓨터**가 네트워크로 연결되어 협력.
  - 각 컴퓨터는 **각자 OS·메모리·저장장치**를 가진 독립된 시스템.
- **특징**
  - 통신은 네트워크(랜·인터넷)를 통해 이루어져 **지연과 장애 가능성**이 큼.
  - 목표: **확장성, 가용성, 장애 허용성**.
  - 예: 클러스터, 분산 DB, Hadoop, Kubernetes, 대규모 웹서비스 인프라.
- **비유**
  - “여러 도시에 있는 팀이 인터넷으로 협업하는 글로벌 프로젝트.”

---

### (3) 비교 요약

| 기준 | 다중처리 | 분산처리 |
|---|---|---|
| 위치 | 한 시스템 내부의 여러 CPU | 여러 시스템이 네트워크로 연결 |
| 자원 구조 | 메모리·OS 공유 | 시스템별 OS/메모리 독립 |
| 통신 | 공유 메모리/버스 → 매우 빠름 | 네트워크 → 상대적으로 느림 |
| 목표 | 성능·병렬 처리 향상 | 확장성·신뢰성·대규모 처리 |
| 장애 | 한 시스템 장애 시 영향 큼 | 일부 노드 장애에도 전체 유지 가능 |

# 📌 2. 컴파일러·인터프리터·목적프로그램

## 2-1. 목적프로그램(Object Program)의 의미

- **정의**
  - 소스코드를 번역한 결과물로, **기계어 또는 기계어에 가까운 형태의 실행 가능한 코드**.
  - 예: `.exe`, `.o`, `.obj`, `.class` 등.
- **역할**
  - 한 번 컴파일해두면, **다시 번역 없이 실행만 하면 되는 완성본**에 해당.
  - CPU나 가상머신이 바로 읽어서 실행할 수 있는 형태.

---

## 2-2. 컴파일러와 인터프리터의 차이

### (1) 컴파일러(Compiler)

- **동작 과정**
  - 소스코드 전체 → 분석·번역 → **목적프로그램 생성** → 목적프로그램 실행.
- **특징**
  - 번역 단계에 시간이 걸리지만, 이후 실행은 빠름.
  - 최적화가 가능해 **성능 향상**에 유리.
- **구조**
  - 소스 → 컴파일러 → 목적프로그램 → 실행.

---

### (2) 인터프리터(Interpreter)

- **동작 과정**
  - 소스코드를 **줄 단위·문장 단위로 읽어 바로 실행**.
  - 별도의 목적프로그램 파일을 남기지 않음(전형적인 인터프리터 기준).
- **특징**
  - 수정 후 바로 실행해 보기 좋고, 디버깅에 유리.
  - 실행 시마다 해석하므로 일반적으로 컴파일 방식보다 느릴 수 있음.

---

### (3) 비교 핵심

| 기준 | 컴파일러 | 인터프리터 |
|---|---|---|
| 목적프로그램 | 생성함 | 일반적으로 생성 안 함 |
| 실행 방식 | 사전 번역 후 실행 | 실행 시마다 해석 |
| 속도 | 실행 빠름 | 상대적으로 느림 |
| 사용 용도 | 성능 중시 프로그램 | 스크립트·실험·교육용 등 |

---

# 📌 3. 네트워크 및 인터넷 관련 개념

## 3-1. ASP와 PHP, 운영체제 연관

- **ASP (Active Server Pages)**
  - Microsoft가 개발한 서버 사이드 스크립트 기술.
  - **Windows + IIS(Internet Information Services)** 환경에서 강세.
- **PHP (Hypertext Preprocessor)**
  - 오픈소스 서버 사이드 스크립트 언어.
  - 전통적으로 **Linux + Apache + MySQL + PHP(LAMP)** 조합이 대표.

> 시험/암기 관점에서  
> **ASP → Windows 계열, IIS 기반**  
> **PHP → Linux 계열, LAMP 스택**  
> 으로 기억하는 경우가 많다(실제로는 교차 사용 가능하나 기본 조합 기준으로 구분).

---

## 3-2. 베이스밴드(Baseband) 전송과 브로드밴드(Broadband) 전송

### (1) 베이스밴드 전송

- **정의**
  - 디지털 신호를 **변조 없이 원형 그대로** 전송하는 방식.
- **특징**
  - 한 채널에 한 신호만 사용(다중화 없음).
  - 거리 제한이 있고, 주로 **근거리 통신(LAN)**에 사용.
- **대표 사용처**
  - Ethernet, 사무실/학교/가정 내부 유선 LAN(UTP, STP 케이블 등).

---

### (2) 브로드밴드 전송

- **정의**
  - 주파수 분할(FDM) 등을 이용하여 **여러 채널을 한 물리 매체에서 동시에** 사용하는 방식.
  - 보통 아날로그 방식의 변조를 사용.
- **특징**
  - 장거리 전송에 적합.
  - 여러 사용자를 동시에 수용 가능.
- **대표 사용처**
  - 케이블TV, 인터넷 백본망, 광역 네트워크(WAN), 통신사 구간.

---

### (3) 핵심 비교

| 기준 | 베이스밴드 | 브로드밴드 |
|---|---|---|
| 신호 방식 | 디지털 직접 전송 | 아날로그 변조·다중화 |
| 채널 수 | 1개 | 여러 개(FDM 등) |
| 거리 | 단거리(LAN) | 장거리(WAN, 통신망) |
| 예 | Ethernet(LAN) | 케이블TV, 통신사 구간 |

---

## 3-3. ADSL과 VDSL

- **ADSL (Asymmetric Digital Subscriber Line)**
  - 비대칭 디지털 가입자 회선.
  - **다운로드 속도 > 업로드 속도**.
  - 가정용 인터넷에서 다운로드 중심 사용 패턴에 최적화.
- **VDSL (Very-high-bit-rate Digital Subscriber Line)**
  - 초고속 디지털 가입자 회선.
  - ADSL보다 **훨씬 높은 전송 속도** 제공.
  - 주로 국사~건물 사이 등 비교적 짧은 동선에서 고속 전송용.

---

## 3-4. OSI 7 계층 요약

| 계층 | 역할(간단) | 대표 프로토콜 | 대표 장비/예 |
|---|---|---|---|
| 7 응용 | 사용자와 직접 맞닿는 서비스 | HTTP, FTP, SMTP, DNS | 웹 브라우저, 메일 클라이언트(소프트웨어) |
| 6 표현 | 데이터 형식 변환, 암호화, 압축 | JPEG, MPEG, SSL/TLS | 소프트웨어 계층 |
| 5 세션 | 연결의 생성/유지/종료 | NetBIOS, RPC, 세션 관리 | 소프트웨어 계층 |
| 4 전송 | 신뢰성·흐름·오류 제어 | TCP, UDP | L4 스위치, L4 방화벽 |
| 3 네트워크 | 라우팅, 논리 주소(IP) | IP, ICMP, ARP, OSPF, BGP | 라우터, L3 스위치 |
| 2 데이터링크 | 프레임 전송, MAC 주소 | Ethernet, PPP, VLAN, STP | 스위치(L2), NIC, 브리지 |
| 1 물리 | 비트 전송, 전기/광 신호 | UTP, 광섬유, RS-232 | 허브, 리피터, 케이블, 모뎀 |

## 3-5. DHCP (Dynamic Host Configuration Protocol)

- **약자**
  - Dynamic Host Configuration Protocol.
- **역할**
  - 네트워크에 접속하는 장비에게 **IP 주소, 서브넷 마스크, 게이트웨이, DNS 서버** 등을 자동으로 할당.
- **장점**
  - 관리자가 수동으로 설정할 필요가 없음.
  - IP 주소를 임대(lease) 개념으로 관리하여 자원 효율성을 높임.
- **동작 흐름(DORA)**
  - Discover → Offer → Request → Acknowledge  
    순으로 클라이언트와 서버가 메시지를 주고받으며 할당.

---

## 3-6. SNMP (Simple Network Management Protocol)

- **약자**
  - Simple Network Management Protocol.
- **목적**
  - 네트워크 장비(라우터, 스위치, 서버, 프린터 등)의 상태를 **모니터링하고 관리**하기 위한 프로토콜.
- **구성 요소**
  - **Manager**: 중앙 관리 시스템(NMS).
  - **Agent**: 각 장비 내에서 동작하는 모듈.
  - **MIB(Management Information Base)**: 관리 정보 구조(트리 형태, OID 사용).
- **주요 기능**
  - CPU·메모리·트래픽 모니터링.
  - 장비 상태/장애 감시.
  - 설정 변경 및 통계 수집.

---

## 3-7. 초기 인터넷 서비스: Archie, Gopher, WAIS, Finger, Intranet

### (1) Archie

- FTP 서버에 있는 **파일 목록을 검색하는 초기 검색 서비스**.
- 웹 검색엔진의 조상격.

### (2) Gopher

- 웹 이전의 **메뉴 기반 텍스트 정보 탐색 시스템**.
- 계층형 메뉴 구조로 문서를 찾아가는 방식.

### (3) WAIS (Wide Area Information Service)

- 광역 정보 검색 시스템.
- 텍스트 기반 데이터베이스/문서 검색 기능 제공.
- 초창기 인터넷 문서 검색 시스템.

### (4) Finger

- 네트워크 상의 **사용자 정보를 조회**하는 서비스.
- 로그인 상태, 이메일, 간단한 사용자 정보 제공.
- 보안 문제로 현재는 거의 사용되지 않음.

### (5) Intranet

- 기업/학교 등 **조직 내부에서만 사용하는 사설 네트워크**.
- TCP/IP, HTTP 등 인터넷 기술을 사용하지만,  
  접근은 내부 사용자로 제한.

---

## 3-8. 테더링(Tethering)과 Wibro

### (1) 테더링

- 스마트폰의 데이터 회선을 이용하여 **다른 기기에 인터넷을 공유**하는 기능.
- 방식: Wi-Fi 핫스팟, Bluetooth, USB 테더링 등.

### (2) Wibro (Wireless Broadband)

- 우리나라에서 개발된 **무선 광대역 인터넷 기술**.
- 이동 중에도 고속 데이터 통신 제공을 목표.
- LTE·5G 등 등장으로 현재는 사실상 사용 중단된 과거 기술.

---

## 3-9. IPS (Intrusion Prevention System)

- **정의**
  - 침입 탐지(IDS) 기능에 더해 **공격을 실시간으로 차단**하는 시스템.
- **역할**
  - 네트워크 패킷을 검사하여 공격 패턴(시그니처, 이상행위)을 탐지.
  - 탐지 즉시 연결을 차단하거나, 세션 종료, 패킷 드롭 등의 조치를 수행.
- **위치**
  - 보통 방화벽 뒤나 내부 구간에 위치하여 트래픽을 깊게 분석.

---

# 📌 4. 그래픽·멀티미디어·압축 개념

## 4-1. 주요 그래픽 기법

| 용어 | 어원/구성 | 의미/용도 |
|---|---|---|
| Dithering(디더링) | dither = 흔들다, 섞다 | 서로 다른 색 점들을 섞어, 팔레트에 없는 중간 색을 표현하는 기법. 저색상 환경에서 색 단계를 부드럽게 하는 데 사용. |
| Interlacing(인터레이싱) | inter + lace = 얽다 | 이미지를 한 번에 모두 표시하지 않고 줄/필드 단위로 번갈아 표시하여, 로딩 중에도 대략적인 화면을 먼저 보여주는 방식. |
| Rendering(렌더링) | render = 표현하다 | 3D/장면 데이터(모델, 조명, 재질)를 입력받아 최종 2D 이미지나 영상을 생성하는 과정. 게임·애니메이션·CG의 핵심. |
| Mezzotint(메조틴트) | mezzo = 중간, tint = 색조 | 인쇄·동판화 기법에서 유래한, 거친 점과 질감으로 중간톤을 표현하는 기법. 그래픽 소프트웨어에서 질감 효과로 응용. |
| Morphing(모핑) | morph = 형태 | 한 이미지/형태가 다른 형태로 자연스럽게 변하는 애니메이션 효과. 얼굴 변신, 로고 변형 등에 사용. |
| Anti-aliasing(안티앨리어싱) | anti + aliasing | 픽셀 기반 계단 현상(aliasing)을 줄이기 위해 경계부를 부드럽게 처리하는 기법. 폰트·곡선·3D 경계를 매끄럽게 만듦. |
| Filtering(필터링) | filter = 거르다 | 블러, 샤프닝, 엣지 검출 등, 이미지의 특정 성분(주파수)을 강조하거나 제거하는 후처리 기법. |
| Modeling(모델링) | model = 모형 | 3D 객체의 형태, 골격, 표면 구조를 만드는 작업. 렌더링 이전 단계로 캐릭터, 배경, 오브젝트 등을 정의. |

---

## 4-2. 무손실 압축(Lossless)과 손실 압축(Lossy)

### (1) 무손실 압축

- **정의**
  - 압축 후 다시 복원했을 때 **원본 데이터와 완전히 동일**하게 재구성되는 방식.
- **특징**
  - 정보 손실 없음.
  - 압축률은 손실 압축보다 낮은 편.
- **예시**
  - ZIP, PNG, FLAC, GIF 등.

---

### (2) 손실 압축

- **정의**
  - 인간이 잘 인지하지 못하는 정보(세밀한 색 차이, 고주파 노이즈 등)를 제거하여  
    **데이터량을 크게 줄이는 방식**.
- **특징**
  - 원본과 완전히 같게 복구 불가.
  - 대신 매우 높은 압축률.
- **예시**
  - JPEG, MP3, AAC, MPEG 계열 동영상 등.

---

### (3) 비교 요약

| 기준 | 무손실 압축 | 손실 압축 |
|---|---|---|
| 복원 | 원본과 100% 동일 | 원본과 일부 차이 |
| 사용 목적 | 원본 보존(문서, 원본 이미지 등) | 전송·저장 효율 최우선(사진, 음악, 동영상) |
| 압축률 | 상대적으로 낮음 | 높음 |

---

## 4-3. MPEG와 손실 압축 사용 이유

### (1) MPEG 약자와 의미

- **MPEG = Moving Picture Experts Group**
- 동영상·음성을 효율적으로 압축하기 위한 **국제 표준 규격을 만드는 전문가 그룹** 및 규격 시리즈.

### (2) 주요 규격

| 규격 | 특징 |
|---|---|
| MPEG-1 | 저해상도 영상, VCD, MP3 포함 |
| MPEG-2 | 디지털 TV, DVD 방송 핵심 규격 |
| MPEG-4 | 인터넷 스트리밍, 모바일 영상에 적합 |
| MPEG-7 | 멀티미디어 콘텐츠 설명·검색(메타데이터) |
| MPEG-21 | 디지털 콘텐츠 분배, DRM 관련 |

### (3) 왜 손실 압축을 주로 사용?

- 영상 데이터의 원본 크기가 매우 크기 때문.
- 예를 들어, 해상도 $1920\times1080$, $24$bit 색상, 초당 $24$프레임이라 하면  
  원시 데이터는 $1$초에 수십~수백 MB에 육박.
- 방송, 스트리밍, 저장을 위해서는 **극단적인 용량 감소**가 필수.
- 인간의 시각·청각 특성을 이용하여 **눈/귀가 잘 못 느끼는 정보부터 버리는 전략**으로 손실 압축 채택.

---

## 4-4. 벡터(Vector) 방식과 비트맵(Bitmap) 방식

### (1) 벡터 그래픽

- **표현 방식**
  - 점, 선, 곡선을 **수학적 함수·기하 정보**로 저장.
- **장점**
  - 아무리 확대해도 깨지지 않고 선명함(해상도 독립).
  - 로고, 아이콘, 폰트, 도형에 적합.
- **고해상도 실사에 부적합한 이유**
  - **실사 이미지의 미세한 질감·텍스처**를 수학식으로 표현하기 매우 어렵고 비효율적.
  - 현실적인 사진 하나를 벡터로 완벽 표현하려면 수식과 객체가 폭발적으로 늘어나 용량·연산량 모두 비현실적.

---

### (2) 비트맵(Bitmap) 그래픽

- **표현 방식**
  - 이미지를 **픽셀 단위의 색상 값**으로 저장.
  - 해상도 $W\times H$, 색 깊이 $b$bit라면 총 데이터량은 대략 $W\times H\times b$에 비례.
- **장점**
  - 실사 이미지, 사진, 동영상, 복잡한 질감 표현에 매우 적합.
- **단점**
  - 해상도가 높아질수록 용량이 급격히 증가.

---

### (3) 저장 공간 측면 비교

- 일반적으로, **같은 내용을 표현할 때**
  - 단순 도형·로고: **벡터가 훨씬 용량이 작음**.
  - 실사 사진·영상: 비트맵 기반이지만, MPEG/JPEG 등의 압축 기법을 적용해 효율화.
- 핵심:
  - **비트맵은 픽셀마다 저장 → 고해상도일수록 용량이 커짐.**
  - **벡터는 수식·명령 집합만 저장 → 해상도와 관계 없이 용량 변화가 적음.**

---

# 📌 5. 보안·개인정보·암호화·바이러스

## 5-1. 개인정보와 간접 정보(평판·평가 정보)

- **개인정보의 법적 정의(핵심 요약)**
  - 생존하는 개인에 관한 정보로서  
    그 정보만으로 또는 다른 정보와 쉽게 결합하여  
    **개인을 식별할 수 있는 정보**.
- **중요 포인트**
  - 이름, 주민번호 같은 “직접 식별 정보”뿐 아니라  
    **평가·의견·행동에 대한 기록**도 특정 개인과 연결되면 개인정보.
- **왜 “타인이 개인에 대해 평가한 간접정보는 개인정보가 아니고 보호받기 힘들다”는 말이 틀린가?**
  - “A는 지각이 잦다”, “4반 반장인 B는 성실하다” 같은 평판 정보도
    - 특정한 개인에 대한 정보이고
    - 다른 정보(반, 직책, 맥락)와 결합하면 개인을 식별 가능.
  - 따라서 **타인의 평가·의견도 개인 식별이 가능하면 개인정보**에 해당하며, 보호 대상이 된다.

---

## 5-2. 공개키 암호화(Public-key Cryptography)의 구조

- **비대칭 키 방식**
  - **공개키(Public Key)**와 **개인키(Private Key)** 한 쌍을 사용.
  - 한쪽 키로 암호화한 것은 다른쪽 키로만 복호화 가능.
- **왜 암호화키(공개키)를 공개하고, 복호화키(개인키)를 비밀로 하나?**
  - 누구나 상대방의 공개키를 이용하여 **안전하게 메시지를 암호화하여 보낼 수 있게** 하기 위해.
  - 예:
    - B의 공개키는 세상에 공개.
    - A가 B에게 메시지를 보낼 때 B의 공개키로 암호화.
    - 복호화는 **오직 B의 개인키로만 가능**.
  - 공개키는 노출되어도 안전하게 설계하고,  
    개인키만 잘 지키면 기밀성 유지가 가능.
- **핵심 구조**
  - 암호화: 송신자 → 수신자의 **공개키** 사용.
  - 복호화: 수신자 → 자신의 **개인키** 사용.

---

## 5-3. 부트 바이러스와 파일 바이러스

### (1) 부트 바이러스(Boot Virus)

- **감염 위치**
  - 디스크의 부트 섹터, MBR(Master Boot Record) 등 **부팅 관련 영역**.
- **특징**
  - 컴퓨터가 켜질 때, OS보다 먼저 실행.
  - OS 재설치만으로는 제거가 어려운 경우도 있음.
  - 초기 플로피 디스크 시절에 특히 성행.

---

### (2) 파일 바이러스(File Virus)

- **감염 위치**
  - 실행 파일(`.exe`, `.com`, `.dll` 등)에 감염.
- **특징**
  - 사용자가 파일을 실행할 때 함께 동작.
  - 감염된 파일이 복사·공유되면서 전파.
  - 일반적인 악성코드 상당수가 이 방식.

---

### (3) 감염 부위 기준 분류

- 부트 바이러스 vs 파일 바이러스는 **“어디를 감염하느냐(감염부위)”를 기준으로 구분**한다.

| 종류 | 감염 부위 | 실행 시점 |
|---|---|---|
| 부트 바이러스 | 부트 섹터/MBR | 부팅 직후(OS 전에) |
| 파일 바이러스 | 실행 파일 내부 | 파일 실행 시 |
