## 1-6. sum을 이용하여 순번 매기기
- &는 +,-와 같은 연산자이므로 지문에 언급이 없더라도 필요에 맞게 사용할 수 있다.

> =TEXT(C30,"yymmdd-")&SUM(($C$30:$C$50=C30)*($F$30:$F$50<F30)*1)+1
> text 함수를 통해 날짜를 정말 원하는 모양으로 만들어버릴 수 있다. 뒤에 -도 붙일 수 있다.
> 배열수식은 배열중에서 조건에 맞는거만 1로하여 선택, 뒤 조건도 그렇게 선택 그리고 선택된 행의 어떤 값으로 더할지 정하는 것이다.
> 오름차순은 해당범위에서 나보다 작은 수들의 수 + 1이 작은 순위이므로 이렇게 구해주어야한다.
> 오름차순: 범위 < 내값 + 1, 내림차순: 범위 > 내값 + 1
> 이렇게 배열수식과 일반 함수를 합쳤을때는 굳이 배열수식으로 묶지 않아도 된다. -> 그냥 enter쳐도 된다.

## 1-7. select case에서 to 문법
```
Public Function fn평가(기관수, 기관점수)
    answer = Round(기관수 / 기관점수, 1)
    Select Case answer
        Case 0 To 0.5
            fn평가 = "미흡"
        Case 0.6 To 1
            fn평가 = "보통"
        Case 1 To 1.5
            fn평가 = "우수"
        Case Else
            fn평가 = "범주 외"
    End Select
End Function
```
> select case에서 0~1범위 이런 걸 표현할때 to 문법을 사용할 수 있다. 이때 case is 가 아닌 그냥 case에 to 문법을 적용하는 것으로 함수를 사용한다.

## 1-8. match 관련 유의 사항
> match의 두번째인자 즉 찾을 범위는 해당 범위가 2줄로 되어있을때 위의 값만 선택해야하며 그 첫번째 값을 기준으로 오름차순 정렬되어있을때는 3번째 인자를 1, 내림차순 정렬되어있을때는 3번째 인자를 -1로 줘야 한다.
> 결과값이 -로 나오는 것은 회계서식이라 그런 것이므로 전혀 문제상황이 아니다. -> 서식은 서식일뿐 시스템적으론 문제가 없기 때문이다.

## 1-9. frequency와 배열수식
> 배열연산을 해서 결과가 열벡터로 나오게 되는 경우는 다 배열수식을 이용한 연산이고 frequency는 보통배열수식이다. 즉 배열수식 연산의 결과는 열벡터 즉 배열이다.
> 이때 frequency는 전체 범위를 드래그하여 실행하므로 채우기 드래그가 없다. 채우기 드래그가 없다면 더 절대화개념도 필요가 없다.
> 시각의 차인지 차의 시각인지 구별하고 문제가 배열수식 문제인지 아닌지도 확인해야한다.
> frequency의 조건값은 결과표의 끝값들을 이용하고 ctrl+shift+enter를 사용하여 확인한다.
