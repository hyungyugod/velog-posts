# 📌 1. 네트워크 기본 개념 정리

## 1-1. 루프백 주소(Loopback)와 localhost
루프백 주소는 네트워크 장치나 외부를 거치지 않고 **자기 자신과 통신하기 위한 특수 주소**이다. 일반적으로 `127.0.0.1`을 사용한다.  
이는 실제 LAN, 라우터를 거치지 않고 OS 내부에서 패킷이 바로 순환되며, **TCP/IP 스택 자체가 정상 동작하는지 테스트할 때 사용**된다.  
즉, 네트워크가 끊겨 있어도 자신과는 통신 가능하다.

`localhost`는 **127.0.0.1을 가리키도록 설정된 도메인 이름**이다.  
hosts 파일에서 다음과 같은 형태로 매핑된다.

```
127.0.0.1   localhost
```

localhost는 이름 체계, 127.0.0.1은 숫자 주소이므로 **둘은 같지만 개념적으로는 구분되는 것**이다.

---

## 1-2. 게이트웨이(Gateway)
게이트웨이는 내부 네트워크 사용자가 **외부 네트워크로 나가기 위해 거치는 첫 번째 경로(출입구)**이다.  
LAN 내부 PC끼리 통신할 때는 경로가 필요 없지만, 외부 인터넷으로 나갈 때는 반드시 게이트웨이를 거쳐야 한다.

예시 흐름:
```
PC → 게이트웨이(공유기) → 인터넷 → 목적지 서버
```

같은 네트워크 대역이면 게이트웨이를 거치지 않고 직접 통신하며,  
다른 대역이라면 반드시 게이트웨이를 경유하여 라우팅된다.

---

## 1-3. ping 명령어
`ping`은 네트워크에서 특정 대상이 **실제로 응답 가능한 상태인지 확인**하기 위한 도구이다.  
기술적으로는 ICMP Echo Request를 보내고 Echo Reply 응답을 받는다.

사용 예:
```
ping 127.0.0.1   → TCP/IP 스택 정상 여부 점검
ping 게이트웨이  → 공유기까지 통신 여부 검사
ping 8.8.8.8     → 인터넷까지 연결 확인
ping google.com  → DNS까지 정상 작동하는지 확인 가능
```

응답시간(ms), TTL 값으로 지연 속도와 경로 깊이를 유추할 수 있다.

# 📌 2. FLOPS & 부동소수점 정리

## 2-1. 부동소수점(Floating-Point)
정수는 소수 표현 불가, 고정소수점은 자리수가 고정된다.  
하지만 과학·AI·시뮬레이션 계산에서는 작은 수, 큰 수를 모두 다뤄야 한다.

부동소수점은 다음과 같은 형태로 표현된다.

```
가수 × 10^지수  →  예:  1.234 × 10^8
```

즉, **소수점이 위치를 옮겨 다니므로 Floating(떠다니는)** 형태가 된다.  
이 덕분에 **넓은 범위의 수 표현이 가능**하며 계산 처리에 적합하다.

---

## 2-2. FLOPS(Floating-point Operations Per Second)
초당 처리할 수 있는 **부동소수점 연산 횟수**이며 CPU/GPU 성능을 비교하는 핵심 지표이다.  
AI나 그래픽 연산처럼 실수 연산이 많은 분야에서 FLOPS 수치가 높을수록 성능이 좋다.

예시:
```
1 TFLOPS = 1초에 1조 번의 부동소수점 연산 처리 가능
```

# 📌 3. CPU 구조: RISC vs CISC, 클럭

## 3-1. RISC와 CISC

### RISC (Reduced Instruction Set Computer)
- 명령어 수를 줄여 CPU가 빠르게 처리하도록 설계
- 구조 단순 → 파이프라이닝 효율이 높음
- 모바일, 임베디드, ARM 구조에 주로 적용

### CISC (Complex Instruction Set Computer)
- 다양한 명령어를 포함하여 **적은 명령으로도 복잡한 작업 가능**
- 초창기 프로그래밍(어셈블리) 난이도를 낮춰줌
- 대부분의 PC, 서버 CPU(x86)가 해당 구조 기반

즉,
```
RISC = 단순, 빠름, 명령 많음
CISC = 복잡, 명령 하나당 기능 큼(초기 개발이 쉬움)
```

---

## 3-2. 클럭(Clock)
클럭은 CPU가 연산을 수행하는 **박자 단위(Hz)** 이다.  
1초에 클럭이 몇 번 발생하는지를 의미하며 1GHz = 10^9Hz.

초창기 컴퓨터는 기계식 계산 장치였고, 톱니바퀴·릴레이의 규칙적 움직임이 존재했다.  
이 규칙적 박동이 Clock이라는 용어로 발전하여 CPU 동작 타이밍 기준이 되었다.

# 📌 4. 메모리 구조

## 4-1. ROM vs RAM

ROM(Read Only Memory)
- 하드웨어 내부에 저장되며 수정이 어려움
- BIOS, 부팅 코드, 펌웨어 저장
- 비휘발성

RAM(Random Access Memory)
- 프로그램·데이터 실행 공간
- 전원 종료 시 내용 삭제(휘발성)

---

## 4-2. DRAM vs SRAM
| DRAM | SRAM |
|---|---|
| Dynamic → refresh 필요 | Static → refresh 불필요 |
| 전하로 데이터 유지 | 플립플롭 구조로 유지 |
| 느리지만 대용량 구현 쉬움 | 빠르지만 비싸고 용량 작음 |
| 메인 메모리 사용 | CPU 캐시 사용 |

# 📌 5. USB & Hot Plugging

USB는 Universal Serial Bus로, 다양한 장치를 연결하기 위한 **범용 인터페이스**이다.

특징:
- **Plug & Play** : 연결 시 자동 인식
- **Hot Plug** 가능 : 컴퓨터가 켜진 상태에서도 연결/제거 가능
- 최대 127개의 장치를 체인 연결 가능

오답 포인트
```
USB 전송 속도 = 12Mbps만 가능 → X
USB 1.1만 해당되는 속도이며 USB2.0=480Mbps 등 규격이 계속 발전함
```

# 📌 6. BIOS / CMOS / Firmware

Firmware = Firm + Software  
→ 하드웨어 내부에서 동작을 제어하는 **내장 프로그램**.

BIOS (Basic Input Output System)
- PC 전원이 켜지면 가장 먼저 실행
- POST(자체 진단) 수행 후 OS 부팅
- ROM/Flash칩에 저장됨 (HDD 아님)

CMOS
- BIOS 설정값 저장 공간
- 배터리로 전원 공급되어 설정 유지

CMOS 오류 발생 원인:
- 배터리 방전
- 설정값 손상
- 메인보드 오류

 증상:
```
시간 초기화, 부팅순서 초기화, CMOS checksum error 표시
```

# 📌 7. 인터럽트 / 채널(DMA)

Interrupt
- CPU가 진행 중인 작업을 잠시 중단하고 외부 요청 처리
- 처리 후 원래 작업으로 복귀하는 메커니즘

채널(Channel)
- CPU 대신 입출력을 처리하고, 완료되면 인터럽트로 CPU에 보고
- CPU 부담을 줄여 시스템 효율 증가

# 📌 8. 디스크 / 레지스트리

## 8-1. 디스크 오류 검사(CHKDSK)
CHKDSK는 파일시스템 논리오류/배드섹터 관리 기능이지 **바이러스 예방 기능이 아니다.**

## 8-2. 레지스트리
레지스트리는 시스템 전체 설정이 저장된 **중앙 데이터베이스**이며  
부팅·설정 파일 묶음이라는 설명은 틀리다.

## 8-3. 조각모음이 여유공간 해결이 아닌 이유
조각모음은 **속도 최적화** 기능이며 파일을 재배열할 뿐, 공간은 늘어나지 않는다.
